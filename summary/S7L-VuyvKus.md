## 鮑里斯·切爾尼：AI重塑程式設計與未來工作模式——來自《Lenny播客》的深度解讀

**主講人：** 鮑里斯·切爾尼 (Boris Cherny)，Anthropic 核心開發者，Claude Code 負責人
**主題：** AI如何徹底改變程式設計行業，Agentic AI 的未來圖景，以及個人應對之道

---

### 一、引言：一場顛覆性的預言

Anthropic 核心開發者鮑里斯·切爾尼在《Lenny播客》專訪中語出驚人地提出：**「程式設計已被基本解決，軟體工程師崗位將消失」**。他詳細拆解了 AI 如何重塑程式設計行業、Agentic AI 的未來圖景，以及普通人應如何應對這場變革。

### 二、鮑里斯·切爾尼與 Anthropic 的「安全使命」

1.  **職業背景與轉折：**
    *   曾任職 Meta，專注程式碼與工程效率研究。
    *   後加入 Anthropic，領導開發了 AI 程式設計領域的標竿工具 **Claude Code**。
    *   **職業插曲：** 曾短暫離開 Anthropic 加入 Cursor 團隊，但兩週後便因 Anthropic 深植於心的「安全使命」而毅然回歸。
2.  **使命驅動：** 鮑里斯強調，這份對「安全」的極致追求，是最初吸引他加入 Anthropic 的根本動力，並貫穿 Claude Code 從誕生到爆發的全過程。他認為，任何產品的酷炫都無法替代使命感帶來的價值。

### 三、Claude Code 的驚人成長與影響力

1.  **爆發式增長：**
    *   發布一周年， Anthropic 估值超過3500億美元。
    *   Claude Code 作為核心增長引擎，一個月內日活躍用戶數翻倍。
    *   **市場佔比：** Semi Analysis 報告顯示，目前 GitHub 程式碼提交中 **4% 來自 Claude Code**，預計年底將達 **20% (五分之一)**。
2.  **生產力革命：**
    *   鮑里斯本人是 Claude Code 的極致使用者。自去年11月以來，他 **100% 的程式碼由 Claude Code 生成**，每天提交10到30個程式碼變更請求，卻未曾親手編輯一行程式碼。
    *   他預估，每位工程師的生產力因此提升 **200%**。
    *   **意義：** 程式設計不再是進入軟體世界的門檻，普通人也能參與構建。

### 四、AI 重構程式設計：從「程式設計師」到「構建者」

1.  **核心觀點：** 「編碼本身已被解決」，工作的核心正從「寫程式碼」轉向「構建」。
2.  **角色轉變：**
    *   AI 接管繁瑣的語法編寫和除錯細節後，工程師的核心樂趣轉為構思產品邏輯、設計系統架構、與用戶溝通需求。
    *   **新稱謂：** 「軟體工程師」這個頭銜將逐漸消失，取而代之的是 **「構建者 (Builder)」**。這些人不再是單一領域專家，而是跨工程、產品、設計等學科的 **通才**。
3.  **歷史類比：印刷機的發明**
    *   14世紀中葉，印刷機打破了少數抄寫員對知識傳播的壟斷，最終催生了文藝復興。
    *   如今，AI 編碼工具如同現代印刷機，將打破程式設計技能的壟斷，讓每個人都具備構建軟體的能力，徹底解放人類創造力。
4.  **鮑里斯的個人經歷印證：** 他大學主修經濟學，自學程式設計最初是為了解決問題（如數學考試作弊），這讓他很早就意識到「程式設計從來不是目的，而是解決問題、構建事物的工具」。AI 接管繁瑣編碼後，他反而找回了程式設計的樂趣。

### 五、Agentic AI 的崛起：從程式設計擴展到所有基於電腦的工作

1.  **Agentic AI 定義：** 不再是簡單的聊天機器人，其核心是「行動」。它能夠使用工具、控制電腦，在現實世界中執行具體任務。
2.  **Cowork 產品：**
    *   **誕生：** 源於觀察到非技術人員「濫用」Claude Code 處理通用任務（如 SQL 分析、基因組分析、數據恢復、醫療影像分析等）。
    *   **功能：** 團隊僅用10天便基於 Claude Code 構建出面向大眾的桌面 Agent Cowork。它能自動處理專案管理、同步電子表格、收發郵件、支付罰單，甚至透過 Chrome 整合啟動瀏覽器、登錄系統、填寫複雜表格 PDF 等。
    *   **鮑里斯的應用：** 他是 Cowork 的重度用戶，用它自動檢查團隊報告、發送 Slack 提醒、同步郵件、回復消息，並並行運行多個任務。
    *   **未來影響：** Cowork 僅是開始，Agentic AI 將徹底改變所有基於電腦的工作方式，影響任何需要使用電腦工具的職業。

### 六、Anthropic 獨特的產品開發哲學

1.  **潛在需求的挖掘：**
    *   偉大創新往往源於用戶對現有產品的「濫用」或意料之外的使用場景（如 Facebook Marketplace 的誕生）。
    *   更深層次：觀察模型試圖實現的目標，給模型搭建最少的「腳手架」，讓它自主決定執行路徑。
2.  **面向未來6個月的模型構建：**
    *   不應被當前 AI 模型能力限制想像，而應預判模型發展方向，為未來更強大的模型構建產品。
    *   **案例：** Claude Code 早期版本僅能完成鮑里斯 20% 的程式碼工作，但團隊堅信模型會持續進化，最終等來 Opus 4 和 Sonnet 4 模型，實現 100% 程式碼生成。這種「押注未來」策略在模型發布時能實現指數級增長。
3.  **不要過度限制模型：**
    *   許多人在構建 LLM 應用時，會設計僵化的工作流程，強制模型按特定步驟執行。
    *   鮑里斯經驗：最佳方式是賦予模型工具和目標，讓它自行規劃執行路徑。
    *   **「苦澀的教訓」呼應：** 更通用的模型總會勝過更專用的模型。與其耗費精力構建複雜工作流，不如押注通用模型的進化，因為腳手架帶來的短期性能提升終將被下一代模型的能力抹平。

### 七、AI 工具使用實戰技巧

1.  **企業應用策略：**
    *   初期不應過度控制工程師的 Token 預算，應給予無限 Token 自由嘗試和創新。
    *   專案證明可行且需規模擴展時，再考慮成本優化（如使用 Haiku 或 Sauna 替代 Opus）。
2.  **開發者核心技巧：**
    *   **始終使用能力最強的模型：** 例如 Opus 4.6。雖然成本更高，但人工糾錯少，整體效率和 Token 成本反而更優。
    *   **善用 Claude Code 的「計畫模式 (Plan Mode)」：** 在提示詞中加入「請先不要寫任何程式碼」，模型會先與你討論方案，大幅提高一次成功率。鮑里斯 80% 的任務都從此模式開始。
    *   **嘗試不同的介面：** Claude Code 支援 iOS/Android 應用、桌面應用、Slack 整合、IDE 擴展等多種形式，選擇最適合自己的工作流。

### 八、Anthropic 對 AI 安全的承諾

1.  **三層安全架構：**
    *   **機制可解釋性：** 深入理解神經網路內部運作方式（如追踪欺騙相關神經元）。Anthropic 大量開源相關工作，推動行業安全發展。
    *   **實驗室評估：** 在可控環境中輸入合成情境，測試模型是否對齊、是否安全。
    *   **盡早發布產品：** 在現實世界中觀察模型行為，獲取真實反饋以持續優化安全性（如 Claude Code 內部使用四五個月後才發布，Cowork 以研究預覽版推出）。
2.  **「力爭上游」原則：**
    *   為 Claude Code 發布開源沙盒環境，確保 Agent 在邊界內運行，無法訪問系統所有內容。
    *   該沙盒不僅支持 Claude Code，還能與任何 Agent 協同工作，旨在推動安全的 Agent 應用普及。

### 九、未來工作場景與人才需求：通才的崛起

1.  **AI 衝擊蔓延：** 首先影響與工程相鄰的角色，如產品經理、設計師、數據科學家。
2.  **角色模糊化與通才化：** Anthropic 內部，設計師大多會寫程式碼，透過 Claude Code 自行解決問題。Cowork 團隊中，產品經理、工程經理、設計師、財務、數據科學家都具備編碼能力。
3.  **未來核心人才能力：** 最強的工程師往往兼具產品思維和基礎設施能力，或擁有出色的設計品味，能深刻理解業務需求並善於與用戶溝通。
4.  **關鍵素質：** 未來幾年最受青睞的人，不僅要熟悉 AI 工具，還要充滿好奇心，能夠跨越多個學科，思考更廣泛的問題。
5.  **失業擔憂的回應：傑文斯悖論**
    *   技術進步會讓能做的事情越來越多，反而創造更多就業機會。
    *   Anthropic 團隊規模在 AI 成為工程工作重要組成部分後持續擴大，Cowork 團隊也在積極擴張。
    *   **歷史類比：** 印刷機的出現並未讓抄寫員失業，反而解放他們時間，專注更具創造性的工作，知識民主化也催生了更多新職業和機會。
    *   **建議：** 與其害怕 AI，不如主動擁抱它，學習使用工具，將自己從繁瑣重複勞動中解放，專注於 AI 無法替代的創造性工作。

### 十、鮑里斯的個人觀點與啟示

1.  **科幻情懷：** 最喜歡中國科幻作家劉慈欣的作品，特別是《流浪地球》短篇小說集，認為其文筆優美，且中國科幻視角獨特。科幻小說也是他加入 Anthropic 的重要契機，讓他思考 AI 技術的發展軌跡。
2.  **AGI 實現後的願望：** 出人意料地想去 **做味噌**。這源於他在日本鄉村生活時學會的慢節奏手工勞作，與快節奏的工程工作形成對比，讓他學會用長遠視角思考問題，在耐心等待中尋求內心平靜。
3.  **人生格言：運用常識 (Use Common Sense)**。他認為，工作中的失敗常源於機械遵循流程而不假思索，真正優秀的結果來自那些從第一性原理思考、善於運用常識的人。
4.  **社交媒體互動：** 近期在 X (原 Twitter) 上更加活躍，直接回復用戶關於 Claude Code 的反饋並快速修復 bug，收穫大量有價值建議。

### 十一、總結：擁抱變革，最大化人類創造力

鮑里斯·切爾尼認為，這場 AI 變革的本質不是取代人類，而是 **解放人類**，讓我們從繁瑣重複的勞動中解脫出來，專注於更具創造性、更有價值的工作。無論是程式碼構建者的崛起，還是 Agentic AI 的普及，都在推動 **人類創造力的最大化**。對於每個人來說，與其恐懼變革，不如主動適應，學習使用 AI 工具，培養跨學科的通才思維，這才是應對未來的最佳方式。
