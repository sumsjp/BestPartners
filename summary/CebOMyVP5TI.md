**訪談摘要：OpenAI 對軟體工程未來的洞見**

**引言：AI 編程的新里程碑**
OpenAI 近日發布了旗艦級 AI 編程模型 GPT-5.3-Codex，標誌著 AI Agent 在成為人類協作者的道路上實現關鍵升級。OpenAI Codex 產品負責人亞歷山大·恩比里科斯在訪談中，深入剖析了這款模型及其對軟體工程工作模式的革命性影響。他強調，其關注點始終是工具能否被全天候使用並真正改變工作方式，而非單純的技術極限。訪談核心議題圍繞一個問題：當工程師將完整任務委託給 AI 時，軟體工程將如何演變？

**一、軟體工程師角色與分工的再定義**

1.  **工程師數量不減反增：** 亞歷山大否定了 AI 會減少工程師數量的判斷，認為未來五年軟體工程師和創造者只會更多。他指出，軟體工程師正站在歷史上又一次被重新定義的門檻上。
2.  **人才棧的壓縮：** 傳統的前端、後端、基礎設施等明確分工在 OpenAI 內部迅速模糊。團隊成員被要求具備更強的全棧能力，並需參與產品設計與判斷。
3.  **角色轉變：** 工程師不再是簡單的代碼執行者，而是「問題的定義者與結果的把關者」。他們的價值體現在「做正確的事」，而非「把事做正確」。
4.  **產品經理角色的探討：** 亞歷山大甚至半開玩笑地提出，在這種新工作模式下，產品經理的角色是否仍然必要值得重新討論。他認為，當 AI 提升工程師的全棧能力和產品思維後，此角色的部分職能可能被融合或替代，尤其在團隊規模不大時。
5.  **人類創造力的聚焦：** AI 帶來的分工壓縮不會削弱人類的創造力，反而會將其聚焦在更有價值的環節。

**二、OpenAI 內部工作模式的革命性轉變**

1.  **GPT-5.2-Codex 為轉捩點：** 在此模型發布前，AI 更多扮演輔助角色（如代碼補全、結對編程）；此後，工作方式發生本質變化，工程師將「整個任務」直接委託給 AI。
2.  **AI 主導代碼生成：** 如今，OpenAI 內部許多工程師幾乎不再使用傳統 IDE，而是全天候運行 Codex。亞歷山大明確表示，絕大多數代碼都是由 AI 寫出來的。
3.  **工程師重心轉移：** 人類工程師的工作重心從代碼實現細節，轉移到任務拆解、計劃評估和結果審查上。

**三、AGI 發展的瓶頸與 OpenAI 的解決方案**

1.  **核心瓶頸：人類驗證與輸入效率：** 亞歷山大提出，制約通用人工智能（AGI）發展的並非算力或模型架構，而是人類驗證工作的速度和輸入效率（即提示詞的撰寫和場景發掘）。人們因「懶得敲太多提示詞」而未能充分利用 AI。
2.  **理想狀態：無感使用與產品化：** AGI 的理想狀態應是「無感使用」，用戶無需琢磨提示詞，AI 便能理解情境並提供協助。
3.  **OpenAI 的發力方向：** 核心是將提示詞和人工操作「產品化」。OpenAI 致力於讓 AI 高度產品化，形式可能是一個對話框、語音輸入，甚至是 AI 自動加入群聊提供幫助。
4.  **個人優先策略：** 他強調，當前最重要的應是為「真實的個人用戶打造工具」，讓每個人都成為被 AI 賦能的「超級個體」，而非過度定制垂直場景工具或優先企業級自動化。應打造開放的創作平台，激發用戶的創造性應用。

**四、Agent 開發的三個階段**

1.  **第一階段：專精軟體工程與編碼：** 利用大語言模型對代碼的天然理解優勢，讓 Agent 在此領域做到卓越（Codex 當前發力點）。
2.  **第二階段：具備操作電腦的能力：** Agent 可在更廣泛的場景中發揮作用。本質上，所有 Agent 都是編碼 Agent，因為編碼是 Agent 使用電腦的最佳方式，其能力可遷移到其他領域。
3.  **第三階段：深度產品化：** 在驗證出有效應用方案後，打造功能高度專一、用戶「開箱即用」的產品。OpenAI 預計在未來幾個月內快速推進這些階段。

**五、企業 AI 落地挑戰與策略**

1.  **落地難題：** 企業在 AI 落地時面臨數據安全、權限配置、合規等障礙，需要完整的 IT 或數據基礎設施團隊支持。
2.  **并行推進策略：** 避免一開始就構建覆蓋所有流程的終極系統，而應「一方面解決系統層面難題，另一方面將 AI 先交到一線員工手中」。讓員工實際使用 AI，建立心理模型，再自然融入工作流。
3.  **OpenAI 的 Atlas 瀏覽器：** 為了解決企業 AI 落地關鍵一環，OpenAI 正構建自有的 Atlas 瀏覽器，以便為企業提供安全、可控的 Agent 瀏覽體驗，使其能訪問尚未開放 API 的內部系統。

**六、效率、留存與 OpenAI 的競爭策略**

1.  **推理速度的重要性：** 推理速度對 Codex 開發者至關重要，但亞歷山大認為這不會形成壟斷，市場競爭將帶來多種解決方案。OpenAI 正從硬件、推理方式、模型層面三管齊下，持續提升效率（GPT-5.3-Codex 響應速度提升 40%，產品整體速度提升 25%）。
2.  **用戶留存與開放策略：** 針對代碼 Agent 之間切換成本低的問題，OpenAI 採取「反直覺」的開放策略，使 Codex 核心執行框架開源，並推動 `agents.md` 成為通用約定。
    *   目前代碼生成任務是廠商中立的，切換成本低。
    *   未來，當 Agent 接入外部系統（如 Sentry, Google Docs）時，連接系統本身將成為高粘性決策。OpenAI 已採用操作系統級沙箱策略，為此做準備。
3.  **OpenAI 的競爭優勢與使命：**
    *   **使命：** 安全地將 AGI 收益帶給全人類，因此不懼競爭，甚至會將模型提供給競爭對手使用，從中學習。
    *   **核心優勢：** ChatGPT 帶來的巨大分發優勢、自研模型與自有執行框架的深度耦合、用戶對新模型的提前訪問權。
    *   **市場決勝關鍵：** 從公司整體看是「算力優勢加最好的模型」並有成功商業模式支撐；從產品層面看是「做出好用、讓人願意用的產品」，堅持服務好個人用戶。
    *   **企業市場策略：** 需要大量的用戶教育、支持複雜配置，並與開發者體驗負責人一起設計工作方式，而非簡單地讓企業「隨便用」。

**七、AI 與人類交互的長期主界面**

1.  **對話界面 + 專用界面：** 亞歷山大認為，最終形態是「以聊天或語音作為通用入口，同時為不同角色打造專用的交互界面」。
2.  **通用入口的易用性：** 用戶無需區分不同 AI，只需與一個「東西」對話即可獲取幫助。
3.  **專用界面的專業效率：** 對於高階用戶，專用界面能提升特定任務的效率（例如，播客準備用聊天，深入代碼用 Codex App）。

**八、AI Agent 市場與 SaaS 顛覆論**

1.  **Agent 市場趨勢：** 最終市場將是「更少的玩家捕獲更多的價值」。未來 Agent 將成為「什麼都能幫你做的超級助手」，企業不會希望有十幾個不同 Agent，而是「一個能處理所有問題的超級 Agent」成為工作重心。
2.  **SaaS 顛覆論的反思：** 亞歷山大認為「SaaS 已死」的觀點被誇大。判斷一家 SaaS 公司是否會被顛覆，關鍵在於：
    *   「這家公司是否真正擁有與人的關係？」
    *   「是否掌握了極其關鍵的系統記錄？」
    *   若答案為肯定，則公司不易消失。如果僅是「膠水層」，則面臨挑戰。
3.  **案例分析：** Salesforce、ServiceNow 股價下跌被市場反應過度。Monday 等公司因其核心需求穩定、簡單且 AI 輔助不划算而具備韌性。
4.  **高風險領域：** 他坦言，客服領域將會是受 AI 強烈衝擊的領域，他本人不願在此賽道。

**九、給未來工程師的建議**

1.  **前所未有的黃金時代：** 當下是成為工程師的最佳時代，AI 工具能大幅降低理解複雜代碼庫、規劃改動和研究的成本。
2.  **稀缺能力：** 正因為構建變得容易，真正稀缺的能力反而更加清晰，包括「主動性、審美以及對品質的執念」。
3.  **核心建議：** 去「構建高品質的東西」。在 AI 時代，一個有思想、有完成度的實際項目，比任何標準化的履歷都更有說服力，更能體現工程師的核心價值。

**總結：**
亞歷山大的訪談勾勒出 AI 編程的未來圖景：工程師的角色將被重新定義，人才棧壓縮，人類創造力將聚焦於更高價值環節。AI 讓寫代碼變得廉價，但同時也讓真正的創造力、主動性和品質執念變得更加珍貴。對於身處科技行業的我們，這場變革已然開始，適應變化並提升核心能力是當務之急。
