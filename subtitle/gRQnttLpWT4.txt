有得也有失

纳德拉还提到，接任微软的CEO

当时，他负责着服务器和工具业务

而且微软将能够向更多的小型企业销售产品
所以整体销售反而会显著增加
作为一名在微软成长起来的内部人士
纳德拉对微软的文化有着深刻的认识
他清楚地记得微软首次成为市值最大的公司时
公司内部弥漫着过度自信的氛围
他深知这种文化可能会导致公司的衰落

无论是古希腊文明还是现代企业

幸运的是，在他成为首席执行官之前

Dweck）所著的关于成长心态的书籍
这本书本来主要是应用在孩子的教育和养育领域

他意识到这种成长型思维正是微软所需要的文化理念
纳德拉将它描述为
从自认为的全知全能者

在他与公司、执行团队或其他人员的每一次会议中

确保这个理念能够深入人心

犹豫不决

如果能够在模型架构上实现非线性突破

在他看来

在微软的历史中

此外
Scaling Laws也是影响微软投资OpenAI的一个重要因素
受到达里奥和伊利亚在OpenAI撰写的关于Scaling Laws的备忘录启发
纳德拉坚信
人工智能技术具有指数级增长的潜力
基于这种信念
微软决定全力以赴地支持OpenAI
随着 OpenAI 在 GitHub、Copilot 等平台上取得初步的成功
微软更是坚定了继续加倍投入的决心
当今的AI竞争格局
已经呈现出了一片多元竞争的趋势
整个科技行业对人工智能的重视程度
达到了前所未有的高度
可以说已经普遍觉醒
这个局面类似于 90 年代末
微软与 SUN 公司竞争时的格局变化
但是如今参与竞争的主体更加的多元化
除了有被称为“Mag 7”的七大科技巨头以外
还有更多的新兴力量不断涌入
在这个竞争舞台上
OpenAI 凭借着强大的技术实力和影响力
可以被看做是“Mag 8”，
它在某种意义上已经成为这个时代
与谷歌、微软或者 Meta 等巨头并肩的重要力量
在这样激烈的竞争环境下
纳德拉认为
未来不太可能出现一家独大的局面
整个世界对于前沿模型的需求是多样化的
需要多个分布在全球各地的提供商
来满足不同的需求
而且每个提供商都会拥有多个不同类型的模型
以及与之配套的应用服务器
就像如今的 Copilot 等应用
实际上就是多模型应用的典型代表
可以预见到的是
未来将会出现专门针对于 AI 应用的全新应用服务器
就如同过去有移动应用服务器和 Web 应用服务器一样
这将会是AI时代应用架构的一个重要发展趋势
而在应用层
由于软件层面的网络效应依然存在
所以将会有更多的参与者涌入
无论是在消费级市场还是企业级市场
不同的网络效应将塑造出多样化的竞争态势
因此，纳德拉常常提醒团队
必须从不同的层级结构出发
深入分析和应对竞争
同时要时刻关注那些
可能突然崛起的新兴创始人和企业
因为在这个瞬息万变的行业中
新的竞争力量随时都可能涌现出来
像OpenAI 就是一个典型的例子
接下来
纳德拉提到了微软如今面临的一些挑战与机遇

微软的传统搜索业务正面临着巨大的变革压力
随着消费者需求的演变
人们不再满足于传统搜索提供的 10 个蓝色链接
而是更倾向于直接获取精准的答案
比如
现在的孩子们在使用互联网的时候
往往会质疑
为什么要花费时间去搜索引擎里查找信息

这种消费习惯的转变
使得Bing等传统搜索业务
面临着严峻的挑战
微软深刻认识到了这个趋势
认为聊天与答案相结合的模式
将是未来搜索业务的发展方向
在这个过程中
分发渠道就成为了关键因素
显然，谷歌在这方面具有明显的优势
因为它在苹果和安卓系统上拥有默认的搜索分发地位

他们正在通过多种方式来积极的应对

他们正在与内容提供者建立紧密的合作

努力推动着生态系统的发展

并且获取数据
这些都是亟需解决的问题
另外，在商业应用场景中
比如 Booking
com 等平台

以微软Copilot为例

在AI时代
这种连接让用户能够更加方便地查询和整合各种数据
在企业 AI 业务方面

微软 AI 业务的主要收入来源是 API 业务

其中，一些非常受欢迎的应用
比如 ChatGPT、Copilot、GitHub Copilot 等

为微软在云计算市场拓展了新的客户群体和业务领域
在传统企业中
微软的 AI 应用也在逐渐普及

虽然这个过程进展相对缓慢

在商业应用领域，纳德拉认为
随着Agent时代的到来
传统商业应用程序的概念可能会发生根本性的变革

本质上是带有大量业务逻辑的 CRUD 数据库

这些业务逻辑将逐渐转移到Agent上

集中在 AI 层进行处理和编排
在这种新的架构下
从Copilot到Agent
再到业务应用的整个流程将实现无缝的衔接
极大地提高企业业务流程的效率和智能化水平
以 Excel 为例

类似于 GitHub 与 Copilot 的结合
为用户带来了全新的使用体验
Copilot 利用 Excel 作为工具
再借助强大的操作空间和 Python 解释器
不仅能够生成数据报表
还能够制定复杂的计划
甚至可以说
Copilot 有能力生成所有的 Excel 内容
这彻底重构了 Excel 的传统使用方式

又或者在进入 Excel 后
Copilot 同样可以根据用户的需求

帮助用户更好地理解和利用数据
这种一体化的工作流程
使得用户能够更加高效地完成各种复杂任务
提高了工作效率和质量
从人力成本与资源配置的角度来看
纳德拉认为通过AI的深入应用
微软的总人力成本会呈现下降趋势

同时
每位研究人员所配备的 GPU 数量也会相应增加
他将人工智能看作是知识工作中的一种精益方法
如同制造业所推行的精益管理理念一样
通过重新审视和优化业务流程
业务流程的重组会以一种全新的、由人工智能驱动的方式回归
而在资本支出与模型扩展方面
这几年
微软的资本支出呈现出显著的增长态势
从 2020 年的大约 200 亿美元
预计将增长到 2025 年的 700 亿美元
而作为超大规模的云服务商

也有着自己独特的模式和策略

在人工智能时代，新的应用需求涌现

只不过，在这个建设过程中

避免陷入不良选择的困境
他们不仅关注供应端的建设
确保有足够的资源来支持业务发展
还会密切留意全球各领域的需求变化
力求实现供应与需求的平衡与多样化
在模型扩展方面
纳德拉坚定地信奉Scaling Laws
他认为，尽管随着集群变得更加复杂
大规模训练的分布式计算问题日益严峻
导致指数增长的难度加大
但是仍然不应该轻易违背Scaling Laws
OpenAI 在这方面一直保持着积极的探索和实践
他们持续进行预训练工作
并且在测试时计算方面取得了重要突破
比如o1模型
这种创新不仅提升了模型的能力
还为人工智能的经济模型带来了新的思路
因为在客户使用 o1 模型的时候
由于采样过程会产生更多的计算资源消耗
而客户需要为此支付费用
从而形成了一种新的盈利模式
不过，纳德拉也清醒地认识到
经济现实对模型扩展的限制

有效地转化为市场销售

此外
还存在着一个所谓“赢家诅咒”的现象

以计算效率更高的方式来实现类似的功能
而无需进行大规模的原始研发投入
这种情况类似于软件的盗版问题
尽管企业可以设定各种使用条款
但是很难完全控制他人对模型能力的提取和利用
因此，他认为在未来
人们对模型扩展的追求将会受到一定程度的限制
企业需要更加理性地权衡在模型能力提升上的投入与产出
尤其是当网络效应更多地开始集中在应用层的时候
更要谨慎考虑在模型层面的大规模投资决策
应该说，与 OpenAI 的合作关系

微软对 OpenAI 的投资不仅是简单的资金注入

使得微软高度关注 OpenAI 的利益
因为 OpenAI 的成功与否
直接关系到微软在AI领域的战略布局和投资回报
从客户服务的角度出发
微软将 OpenAI 当作一个重要的大客户来对待
致力于为它提供优质、高效的服务
满足它在技术、资源等多方面的需求
在业务合作层面
双方在多个领域存在着合作与竞争的复杂关系
以 API 为例，对于Azure 客户来说
使用Azure 上的OpenAI API是一个自然的选择
但是
对于那些在 AWS 等其他云平台上的客户
可能就会直接选择 OpenAI 的 API
在合作过程中
双方也面临着一些挑战和紧张的关系
OpenAI 追求快速发展和技术突破的节奏
与微软所需要遵循的纪律以及整体限制之间
需要进行精细的平衡和协调
在模型训练方面
微软基于与 OpenAI 的合作关系以及对知识产权的考量
也采取了独特的策略
微软不会进行重复的、无意义的模型训练工作
而是选择集中计算资源
将重点放在提升模型的适应性和特定能力上
比如
微软会在符合一定原则的基础上开展预训练工作
同时积极地开发多样化的模型权重和模型类别
来满足市场上各种复杂的应用场景需求
对于 OpenAI 的一些重大决策
比如涉及到战略转型、盈利模式调整等关键问题
微软虽然作为投资者拥有一定的话语权
但是最终决策权仍然掌握在 OpenAI 的董事会和核心团队手中
微软表示会积极支持 OpenAI 做出的任何决策
因为他们深知OpenAI 在当前人工智能行业中的标志性地位
从长远来看
一个强大而且成功的 OpenAI
也符合微软自身的利益
因此
微软希望能够与 OpenAI 保持长期稳定的合作关系
通过双方的共同努力
在AI领域实现更大的突破和发展
最后，在AI安全与开源闭源的问题上
纳德拉认为
开源与闭源在AI领域是两种不同的策略
它们并非是相互对立的宗教式的阵营
而是具有各自独特价值和应用场景的两条创新之路
以 Meta 为例
Meta 试图开源Llama模型的做法
就有点类似于 Linux 的发展模式
Linux 作为一个开源操作系统
通过众多实体的共同参与和贡献
逐渐形成了一个庞大而活跃的生态系统
微软实际上也是 Linux 生态系统的重要贡献者之一
这表明不同的企业在开源模式下可以实现互利共赢
当然
闭源模式也具有一定的合理性和必要性
微软自身就拥有许多闭源产品
这些产品在特定的市场需求和商业环境下
能够发挥出独特的优势
但是
无论是开源还是闭源的人工智能系统
都需要遵循相关的法规要求
安全标准将平等地适用于两者
不能简单地认为开源系统就一定比闭源系统更加安全
反之亦然
在资本主义的商业环境下
多样化的竞争格局对大家来说是更为有利的
不同的公司会根据自身的战略定位、技术实力、市场需求等因素
自主选择适合自己的开源或者闭源道路
同时
政府在人工智能的发展过程中将扮演越来越重要的监管角色
无论是在模型的研发、应用还是数据安全等方面
都将会出台一系列严格的政策和法规
未来
专门的 AI 安全机构也将会应运而生
负责制定和执行统一的安全标准
确保人工智能技术的健康、可持续发展
在国家安全层面
人工智能已经引起各国政府的高度关注和重视
政府在人工智能模型的发展方向、监管制度的设计等方面
拥有着广泛的话语权
他们的决策也必将深刻影响人工智能产业的未来走向
