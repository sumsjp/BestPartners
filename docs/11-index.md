**Google Cloud Next 大會深度解析：AI 落地企業核心業務的五層藍圖與智能體操作系統**

### **一、 核心觀點與戰略投入**

*   **定調 AI 落地：** Google Cloud CEO 托馬斯·庫里安（Thomas Kurian）明確指出，AI 落地已從「個案項目實驗」進化至「企業日常運轉的一部分」。
*   **戰略投入規模：**
    *   Google 母公司 Alphabet 的資本支出預計從 2022 年的 310 億美元，飆升至 2026 年的 **1750-1850 億美元**，四年增長近 6 倍。
    *   其中，2026 年超過一半的機器學習計算投入將直接流向 **Google Cloud 業務**。
    *   這是一場「傾全公司之力，押注未來十年基礎設施」的戰役。
*   **內部 AI 應用成效：**
    *   **程式碼生成：** Google 內部 75% 的新程式碼由 AI 生成（半年前僅 50%）。
    *   **複雜程式碼遷移：** 多角色智能體系統將規劃、編排、執行分離，速度比一年前快 6 倍。
    *   **市場行銷：** 大模型批量生成創意變體，工作效率提升 70%，轉化率提升 20%。
    *   **安全營運：** 智能體每月自動分類數萬份威脅報告，威脅緩解時間縮短 90% 以上。
*   **AI 階段的挑戰：** CEO 桑達爾·皮查伊（Sundar Pichai）指出，當前 AI 處於「創新週期中那個凌亂的中間階段，基礎構件正在匯聚，但整個體系仍然混亂」。
*   **Google Cloud 的使命：** 將這種「混亂變成秩序」，將「零散的能力變成一套可規模化落地的企業系統」。

### **二、 「智能體企業」五層藍圖 (The Agentic Enterprise Blueprint)**

托馬斯·庫里安提出，Google 智能體企業藍圖由五層嚴密架構組成，貫穿「智能」與「自動化」的價值交付：
> **智能 + 自動化 = 交付價值**
> （智能來自數據，自動化由智能體驅動）

這套系統旨在解決企業如何管理、調度、安全運行成千上萬個智能體的問題。

**五層架構概述：**

1.  **AI 超級電腦 (AI Hypercomputer)：** 最底層的 AI 運算基礎設施。
2.  **智能體數據雲 (Agentic Data Cloud)：** 提供可信的數據上下文。
3.  **智能體安全防禦 (Agentic Defense)：** 機器速度的安全防禦體系。
4.  **Gemini 企業智能體平台 (Gemini Enterprise Agent Platform)：** 智能體全生命週期管理與編排中樞。
5.  **智能體特遣隊與 Workspace 智能層 (Agentic Task Force & Workspace Intelligence)：** 直接面向員工與客戶的應用層。

### **三、 生態系統與合作**

Google Cloud 在大會上展示了龐大的 AI 生態版圖：

*   **Apple：** Google Cloud 成為 Apple 首選雲提供商，雙方將基於 Gemini 技術共同開發下一代 Apple 基礎模型，以支援更個性化、更智能的 Siri。
*   **NVIDIA：** Google Cloud 將是全球首批提供 NVIDIA Vera Rubin NVL72 算力的雲廠商之一。
*   **Anthropic：** 平台將正式接入 Anthropic Claude Opus 4.7，與 Google 自家的 Gemini 3.1 Pro 並行提供服務。
*   **Microsoft 365：** Gemini Enterprise 新增與 Microsoft 365 的互操作能力，Canvas 中創建的文檔可直接導出為 Office 格式。

### **四、 逐層深度解析五層藍圖**

#### **第一層：AI 超級電腦 (AI Hypercomputer)**

*   **核心理念：** 「在智能體時代，計算不再由單顆芯片定義，計算就是整個數據中心。」 – 阿明·瓦赫達特（Amin Vahdat）
*   **關鍵創新：第八代 TPU**
    *   **里程碑：** 谷歌 TPU 發展十年來，首次將訓練 (TPU 8t) 和推理 (TPU 8i) 分離為兩顆獨立芯片。
    *   **TPU 8t (訓練芯片)：**
        *   為大規模模型訓練優化，提供 121 EFLOPS 的 FP4 算力，比上一代提升 2.8 倍。
        *   SuperPod 可擴展至 9600 顆芯片，芯片互連帶寬翻倍。
    *   **TPU 8i (推理芯片)：**
        *   為推理場景量身定制，支援數百萬並發智能體近乎零延遲運行。
        *   提供 11.6 EFLOPS 的 FP8 算力，比上一代提升 9.8 倍。
        *   集成了專用集合通信加速引擎，降低芯片延遲 5 倍。
    *   **供應鏈：** TPU 8t 由博通設計，TPU 8i 由聯發科設計，英特爾、Marvell 和台積電參與供應鏈。
*   **基礎設施升級：**
    *   **儲存：** Managed Lustre 支援高達 10TB/s 吞吐量。
    *   **網路：** 全新的 Virgo Network 可串聯 134,000 顆芯片，提供 47Pb/s 無阻塞帶寬，總算力達 170 萬 EFLOPS。
    *   **通用計算：** Axion N48 計算實例，基於定制 Arm CPU，提供最高 2 倍性價比及 80% 更優每瓦性能，為智能體提供持續運行底座。

#### **第二層：智能體數據雲 (Agentic Data Cloud)**

*   **核心理念：** 「沒有上下文的推理只是猜測...你承擔不起猜測的代價。」 – 卡爾蒂·納拉因（Karthik Narain）
*   **關鍵創新：打造「可信上下文」(Trusted Context)**
    *   **Knowledge Catalog：**
        *   企業通用上下文引擎，原生集成 BigQuery。
        *   Smart Storage 自動標記、增強、準備非結構化數據供智能體使用，無需數據工程。
        *   Gemini 驅動，自主提取實體、映射關係、學習企業專屬業務語義。
        *   零拷貝直接訪問 Palantir、Salesforce、SAP 等主流企業應用。
        *   與 Gemini Enterprise 的 Deep Research Agent 深度集成，執行多步推理，返回帶來源的商業答案。
    *   **Data Agent Kit：** 將 AI 數據科學能力直接集成到開發者工作環境 (如 VS Code)，透過自然語言聲明意圖即可構建數據管道和部署模型。
    *   **Lightning Engine for Apache Spark：** 提供領先的 Spark 性能，性價比是市場領導者的 2 倍。
    *   **Cross-Cloud Lakehouse：** 基於 Apache Iceberg 標準，分析引擎可直接推理任何雲 (AWS, Azure, SaaS) 中的數據，無需移動數據，消除供應商鎖定。
*   **價值展示：** 現場演示中，評估虛擬產品全球上市僅需 5 分鐘（傳統方式需 5 週），高效整合多源數據並提供決策。

#### **第三層：智能體安全防禦 (Agentic Defense)**

*   **核心挑戰：** 網路攻擊已進入「機器速度時代」，人類安全團隊反應不及。
    *   漏洞平均利用時間降至「負 7 天」。
    *   攻擊者從入侵到轉移權限僅需 22 秒。
*   **谷歌雲安全戰略：**
    *   **Gemini 原生智能體安全運營中心 (Agentic SOC)：**
        *   分類智能體將 30 分鐘調查縮短至 60 秒。
        *   威脅狩獵與檢測智能體主動掃描風險，速度和規模超越人類。
        *   結合 Mandiant、VirusTotal、Chrome 和暗網情報，以 98% 準確率識別外部威脅。
    *   **Wiz (雲安全平台，Google 斥資 320 億美元收購)：** 定位為全球首個 AI 應用保護平台 (AI-SPP)，解決 AI 時代四大安全挑戰。
        *   **Wiz 智能體體系：**
            *   **紅色智能體 (Red Agent)：** 像友好黑客，持續掃描外部暴露面，驗證風險。
            *   **綠色智能體 (Green Agent)：** 自動化風險分類與修復流程，定位代碼源頭，將修復方案推送給開發者或編碼智能體。
        *   德索薩（Francis deSouza）強調：「我們已徹底閉合了防禦者與構建者之間的循環。」
    *   **影子 AI 防禦：** 警告企業內「影子 AI」（未經授權的模型和智能體）是最大安全威脅，凸顯智能體管理和安全的複雜性。

#### **第四層：Gemini 企業智能體平台 (Gemini Enterprise Agent Platform)**

*   **核心定位：** 「智能體企業的任務控制中心」，解決如何管理數千個智能體的問題。
*   **模型矩陣：**
    *   **Gemini 3.1 Pro：** 針對複雜工作流編排優化。
    *   **Nano Banana 2：** 高保真視覺素材生成。
    *   **Veo 3.1 Lite：** 成本最優視頻模型。
    *   **Lyria 3 Pro：** 企業級音頻和音樂生成。
    *   同時接入 Anthropic Claude Opus 4.7。
*   **智能體工具鏈：**
    *   **Agent Studio：** 透過自然語言構建和部署智能體。
    *   **Agent Registry：** 統一控制點，索引企業內部智能體和工具。
    *   **Skills and Tools Registry：** 定義模塊化指令包，暴露所有 GCP 和 Workspace 服務能力接口。
    *   **Agent Marketplace：** 搜索和部署來自 Atlassian、Box、Oracle 等合作夥伴的專業智能體。
*   **關鍵基礎設施：原生集成 MCP**
    *   Google 將所有 GCP 服務暴露為 MCP 服務器，實現雙向打通。
    *   任何兼容 MCP 的智能體均可與 Google Cloud 服務交互，反之亦然，徹底打破智能體生態壁壘。
*   **安全與編排能力：**
    *   **Agent-to-Agent 編排：** 智能體之間無縫委派任務。
    *   支援生成式與確定性兩種編排模式，確保關鍵工作流可追溯。
    *   每個智能體擁有唯一的加密身份憑證 (Agent Identity)，全程可追踪、可審計。
    *   **Agent Gateway：** 集中管理，執行安全策略。
    *   **Model Armor：** 模型與企業數據間的防護層，防止敏感數據洩露。
    *   **Agent Observability：** 符合 OpenTelemetry 標準，可視化智能體執行路徑。
*   **智能體記憶問題：** Gemini Enterprise Projects 為智能體提供高保真專屬工作空間，運用 Deep Think 能力解決複雜業務挑戰，避免上下文污染。
*   **價值展示：** 現場演示中，一家全球家具零售商的運營人員僅需一個提示詞，多個智能體在數分鐘內協同完成市場調研、數據洞察、產品策略、任務創建、演示文稿生成等一系列工作。

#### **第五層：智能體特遣隊與 Workspace 智能層 (Agentic Task Force & Workspace Intelligence)**

*   **核心目標：** 直接面向員工與客戶，消除「上下文稅」（在多個應用間切換造成的浪費）。
*   **Workspace Intelligence：**
    *   統一智能層，利用 Gemini 推理能力連接 Gmail、Docs、Sheets、Slides、Chat 等所有信息節點，打破信息孤島。
    *   將 Google Chat 變成工作指揮中心，透過 Ask Gemini 快速定位緊急事項、文檔，自動生成完整的 Google Slides 演示文稿。
    *   新增微軟 365 互操作性，Canvas 文檔可直接導出為 Office 格式，將 Microsoft 365 遷移到 Google Workspace 的速度最高提升 5 倍。
*   **Agentic Task Force：**
    *   預建的專業化智能體，將智能體定義為「業務的戰略延伸」。
    *   核心產品包括購物智能體、食品訂購智能體、客戶互動智能體。
    *   全新的全渠道網關 (Omnichannel Gateway) 確保智能體在網頁、移動、語音等不同渠道間保持上下文連貫。

*   **霍尼韋爾 (Honeywell)：** 運用超過百萬產品規格訓練數字孿生，為建築管理生成數十億條洞察。

Google Cloud Next 2026 大會發布的，不僅僅是一系列新產品，而是一套**完整的企業智能體操作系統**。從 1800 億美元量級的基礎設施投入，到第八代 TPU 的芯片革命，從可信數據上下文的構建，到機器速度的安全防禦，從統一的智能體管控平台，到每一位員工都能用的生產力升級，五層藍圖層層咬合，旨在徹底解決企業 AI 從試點到規模化的所有核心瓶頸。

## 霍華德·馬克斯沃頓對話核心思想：穿越周期的投資哲學

### 引言

近日，全球知名投資大師、橡樹資本（Oaktree Capital）聯合創始人霍華德·馬克斯（Howard Marks）應邀在其母校賓夕法尼亞大學沃頓商學院，與金融學兼職教授克里斯托弗·蓋齊（Christopher Geczy）展開一場深度對話。此次交流中，馬克斯深入剖析了關於市場情緒、買賣策略、成長與價值辨析以及風險控制的底層投資思維。

本次對話前，主辦方特別要求學生研讀了馬克斯的三篇核心備忘錄：《少一點虧損，還是多一點贏家？》、《什麼才真正重要？》以及《測量市場溫度》。這三份材料共同構成了他一套完整且歷久彌新的投資哲學。本文將結合2026年當前的市場環境，探討這套哲學的實戰應用。

### 一、市場情緒：當前我們身處何方？

霍華德·馬克斯對當前市場的判斷是：**市場情緒已從極度樂觀向負面方向出現明顯擺動，且這種擺動既劇烈又突然。然而，整體市場仍被樂觀情緒所支撐，遠未達到絕望或自暴自棄的程度。**

得出此結論的原因如下：
*   **長期上漲積累的樂觀情緒：** 過去很長一段時間，全球市場持續上行。標普500指數在2023、2024、2025年分別上漲約26%、27%和18%，三年累計漲幅達87%，若從2022年第四季度低點算起，整體漲幅已超100%。這種連續大漲本質上由投資者的樂觀情緒推動。
*   **認知失調現象：** 在樂觀市場中，投資者易出現認知失調，傾向於吸收好消息、忽略壞消息，正如摩根大通CEO傑米·戴蒙所言，人們常會假裝看不見第一隻“蟑螂”。
*   **市場轉向的觸發點：** 情緒鐘擺不會永駐一端，負面因素終將累積。馬克斯梳理了此次市場轉向的幾個關鍵觸發點：
    *   **關稅政策衝擊：** 去年4月，特朗普政府宣布超預期新一輪關稅，導致市場一周內快速下跌約15%。
    *   **AI技術對軟件行業的顛覆：** 今年2月，OpenAI和Anthropic新一代編程模型的發布，直接衝擊傳統軟件公司的核心價值，引發對其巨額信貸債務風險的擔憂。
    *   **私人信貸領域風險暴露：** First Brands和TriColor等公司破產並伴隨潛在欺詐，引發市場對私人信貸放貸標準過低的質疑。
    *   **地緣衝突：** 加劇能源與經濟不確定性，進一步推升投資者擔憂。
*   **人性的極端化：** 現實世界的事態常在“還不錯”和“沒那麼好”之間小幅擺動，但投資者的判斷卻容易從“完美無缺”跳到“毫無希望”。
*   **估值偏高：** 儘管經歷回调，標普500市盈率約22倍，仍高於16-17倍的歷史均值。常見的“這次不一樣”論調，在歷史上多次泡沫中均出現過。

**結論：** 當前市場處於樂觀退潮、謹慎升溫的階段，但遠未達極端低估。

### 二、買入時機：何時才是“抄底”的最佳時機？

**霍華德·馬克斯的結論直截了當：投資中最愚蠢的做法之一，就是非要等到底部再買。**
*   **“底部”的不可預測性：** 所謂底部，是資產價格開始上漲前的最後一天。事後才能知曉，事前無法預判和精準捕捉。追求完美地等待底部，往往會錯失整個行情，眼睜睜看著便宜資產漲回合理價格。
*   **買入的唯一合理依據：** 判斷買入的唯一合理依據是“東西已經便宜了”。這可通過估值、基本面和安全邊際來判斷，而不是去判斷今天是否是下跌的最後一天。
*   **2008年金融危機的實踐：** 在2008年全球金融危機中，當金融體系瀕臨崩潰時，橡樹資本的決策是：如果世界真的熔毀，做什麼都沒意義；如果世界沒垮，則必須出手。他們買入的是被杠杆收購公司最優先級債務，安全邊際極高，即使公司價值僅剩原有四分之一，仍不會虧損。在市場恐慌、價格低到極致時，他們逆勢大舉投入，最終獲得豐厚回報。
*   **逆勢投資的前提：**
    1.  **有錢可投：** 資金需在危機前募集到位，而非危機來臨時才籌集。
    2.  **有膽量去投：** 需具備對抗全場恐慌情緒的勇氣。
    3.  **危機前保持謹慎：** 平時不盲目放大風險，才能在機會來臨時有能力布局。

### 三、投資視角：成長與價值，並非水火不容

霍華德·馬克斯明確否定了將成長投資（買高增長科技股）與價值投資（買低估值傳統股）視為對立關係的機械二元劃分。
*   **歷史誤解：** 20世紀60年代前，股票市場並無成長與價值之分。1962年後，“成長股”概念出現，指未來前景光明的公司。隨後，不投成長股的人自稱為“價值投資者”，才固化了成長=高增長、價值=低價格的錯誤劃分。
*   **核心在於價格與價值的匹配：** 增長快慢不代表投資的聰明與否。關鍵在於**“你為這種增長支付了什麼樣的價格”**。高增長不等於無限溢價，低增長也不代表天然有價值。價格與內在價值的匹配度，才是決定投資成敗的核心。
*   **“漂亮50”泡沫的教訓：** 1969年，市場追捧基本面優異的“漂亮50”成長股，但因價格嚴重高估，即使持有五年，最終也虧損約95%。這證明了任何資產都不可能貴到不能買，也沒有任何資產糟到只要價格足夠低就不能成為好買賣。**“好的投資，不是來自於買的好東西，而是來自於東西買得好。”**
*   **當前科技股投資應用：** 無論是追捧AI巨頭、科技七巨頭，還是完全排斥高成長只盯低估值藍籌，都可能走向極端。正確的做法是，對所有股票都要評估其內在價值，只在價格低於價值時買入，並為增長支付合理的成本。

### 四、賣出策略：何時放手，何時堅守？

市場上關於賣出的討論遠少於買入，大多數投資者的賣出決策受情緒驅動。
*   **情緒化賣出的非理性：** 漲了怕利潤回吐而賣出，跌了怕繼續虧損而賣出，這兩種行為不可能同時合理，完全是被情緒牽引。
*   **理性賣出法則：反向買入。** 假裝自己目前未持有該資產，問自己一個問題：“如果是今天，我還會買它嗎？”
    *   **如果答案是“不會”：** 說明當前價格缺乏吸引力或基本面惡化，應考慮賣出。
    *   **如果答案是“會”：** 則應繼續持有。
*   **“持有”是一種中間狀態：** 當資產價格合理、公司基本面持續改善，既未便宜到非買不可，也未貴到必須賣出時，安心持有就是最佳選擇。
*   **警惕過早賣出偉大的資產：** 馬克斯基於自身謹慎性格，曾錯過巨大收益。他以亞馬遜（Amazon）為例，從1999年泡沫時的90美元跌至6美元，隨後復權漲至3300美元。大多數人會在小幅上漲後賣出鎖定收益，卻錯過了85%以上的複利增長。
*   **賣出核心：判斷是否仍是稀缺的複利機器。** 真正偉大的投資想法極度稀缺，普通人一生遇不到幾個。賣出決策不應盯著短期漲跌，而是判斷資產的核心邏輯是否依然成立，它是否仍是能持續創造價值的標的。只要核心邏輯未變，就不要輕易拋售。

### 五、投資哲學：求勝致勝，抑或少犯錯誤？

投資的終極選擇是：你想靠更多制勝的辦法賺錢，還是靠盡量少犯致命錯誤來盈利？這個問題沒有標準答案，但決定了投資體系的長期可持續性。
*   **網球比賽類比：**
    *   **職業網球：** 贏球者必須主動進攻，打出更多制勝球。
    *   **業餘網球：** 贏球者往往是失誤最少的，穩定把球打回對面，等待對手犯錯。
*   **投資領域的應用：**
    *   **信貸/固定收益投資：** 核心是“少犯錯，避開壞賬”，靠不虧來賺取穩定回報，屬於“贏得輸家的遊戲”。
    *   **風險投資/成長股投資：** 必須抓住幾個“超級大贏家”，以少數制勝之舉來覆蓋成本並獲得超額收益。
*   **選擇適合自己的道路：** 霍華德和橡樹資本選擇的是“少犯錯”的路線，將風險控制置於首位，追求業績的一致性和穩定性。但這並非唯一正確的路徑。投資者必須選擇一條**適合自己的性格、能力圈和風險承受能力**的道路，而非盲目模仿他人。模仿職業選手的業餘選手會頻繁失誤，風險承受能力低的普通投資者盲押高風險項目也可能虧損慘重。

### 總結與啟示

霍華德·馬克斯此次對話的核心內容可濃縮為以下五點，以助記憶和實踐：

1.  **測量市場溫度：** 市場情緒永在極端之間擺動，勿被樂觀或恐慌裹挾，需理性判斷估值高低。
2.  **放棄抄底幻想：** 買入的唯一依據是資產足夠便宜，安全邊際比精準擇時更重要。
3.  **價格匹配價值：** 成長與價值並非對立，好投資不是買“好東西”，而是“買得好”，價格永遠要與價值匹配。
4.  **珍稀複利機器：** 賣出不看短期漲跌，而是判斷當前是否仍願重新買入。珍惜稀缺的複利機器，勿過早下車。
5.  **找到自己的路：** 投資是一場適合自己的遊戲，要么靠少犯錯穩贏，要么靠制勝之舉突破，找到屬於自己的路比追逐短期收益更重要。

霍華德·馬克斯的投資哲學，沒有複雜模型，也沒有暴利技巧，全是關於如何對抗人性、尊重周期、堅守常識。在科技快速迭代、市場波動加劇的今天，這些思維比任何短期技巧都更具價值。投資的終極目標並非一夜暴富，而是在不確定的市場中，長期、穩定、可持續地積累財富。

## Cerebras Systems：AI推理芯片的顛覆者與其IPO之路

### 核心摘要

本報告深入剖析了Cerebras Systems這家在AI算力領域挑戰英偉達霸權的公司。Cerebras憑藉其獨特的晶圓級芯片技術，專注於AI推理加速，並在性能上超越英偉達最新旗艦產品。公司已遞交IPO申請，預計2025年實現盈利，估值高達230億美元，並與OpenAI、AWS等巨頭建立戰略合作。報告將從公司概覽、技術壁壘、商業化進程、競爭格局及行業洞察等多維度，呈現Cerebras如何將一個75年未解的技術幻想變為商業現實，並有望改寫AI芯片的全球格局。

### 一、 公司概覽與IPO亮點

*   **公開上市計劃：** Cerebras Systems於2026年4月17日向美國SEC提交IPO申請，計劃在納斯達克上市，代碼為CBRS。
*   **驚人財務預期：**
    *   預計2025年全年營收5.1億美元，淨利潤8790萬美元，實現從巨額虧損（2024年淨虧損4.848億美元）到盈利的逆轉。
*   **高額估值與投資方：**
    *   在2026年2月完成的H輪融資中募資10億美元，估值達到230億美元。
    *   投資方陣容豪華，包括老虎環球、AMD、富達、Benchmark Capital、Altimeter等頂級機構。
*   **OpenAI超級計算協議：**
    *   與OpenAI簽署的計算協議規模可能從超100億美元擴展至300億美元。
    *   OpenAI還將獲得Cerebras最多10%的股權認購權證。這標誌著頂級AI模型公司開始大規模押注非英偉達體系的芯片廠商。

### 二、 技術創新與核心壁壘

*   **專注AI推理加速：** Cerebras的核心定位是AI推理加速賽道，專為大語言模型推理場景做極致優化。
*   **全球最大芯片：** 製造出人類歷史上面積最大的芯片，達到傳統GPU的56倍。
*   **性能卓越：**
    *   CEO安德魯·費爾德曼聲稱，其芯片推理速度比英偉達GPU快數十倍。
    *   獨立機構Artificial Analysis實測（2025年5月，Meta Llama 4 Maverick 400B模型）：Cerebras芯片達到2522 token/秒，約為英偉達最新Blackwell B200（1038 token/秒）的2.4倍。
*   **核心技術判斷：**
    1.  **數據移動是瓶頸：** AI計算的核心矛盾不在於算力不足，而是數據在處理單元之間移動的成本太高。傳統處理器架構不適應AI模型高頻中間結果傳輸。
    2.  **推理内存訪問模式：** AI推理需密集讀取内存。一個70億參數模型生成一個詞，就需要從内存搬運約140GB數據，GPU的HBM内存雖容量大但讀寫速度慢，形成瓶頸。
*   **Cerebras的解決方案——晶圓級芯片與SRAM：**
    *   直接將芯片做得足夠大，可在芯片上集成足夠多的SRAM（44GB）。
    *   結合SRAM的極致速度優勢與大容量，解決容量與速度問題。
    *   數據幾乎不需要離開芯片，大幅降低IO數據傳輸帶來的功耗。
*   **解決晶圓級芯片良率難題：**
    *   晶圓級芯片良率一直是75年來行業公認的難題（缺陷率高）。
    *   Cerebras首次將存储器行業的冗餘策略移植到處理器芯片：將處理器設計成數十萬塊相同的小單元(tile)，缺陷單元直接關閉，預留的冗餘單元自動頂上，完美解決大芯片良率問題。
    *   經過近五年的反覆失敗與重新設計，Cerebras的系統於2019年底成功運行，並在2026年實現晶圓良率超越英偉達同類產品。

### 三、 創始人與創業理念

*   **安德魯·費爾德曼 (Andrew Feldman)：** 62歲的連續創業老兵，前四家公司均成功退出，擁有成熟的硬件創業及商業化經驗。
*   **成長環境與創業基因：** 在斯坦福校園長大，父母皆為教授，耳濡目染，燃起創業熱情。
*   **商業理念：**
    *   “基礎設施”思維：相信為他人鋪路，應享受其帶來的回報。
    *   堅信“更快更便宜的技術”：總能讓市場變得更大，而非更小，就像寬帶普及後誕生的Netflix和亞馬遜轉型。
*   **與OpenAI的淵源：**
    *   2015年，Feldman與聯合創始人會見了當時還名不見經傳的Sam Altman、Ilya Sutskever等人。
    *   OpenAI團隊堅信AGI將在幾年內實現，這堅定了Feldman對AI計算需求將爆發式增長的判斷，並認為需要一種與現有GPU完全不同的全新架構。
    *   Feldman坦言這是他五次創業以來，第一次嚴重低估市場規模，AI算力增長速度遠超預期。

### 四、 商業化進程與客戶拓展

*   **早期核心支點：** 阿聯酋G42集團。
    *   雙方合作訓練了阿拉伯語領域領先的語言模型和基因序列模型。
    *   為Cerebras帶來數億美元設備訂單，一度佔其全年營收的87%。
    *   在此合作中，Cerebras鍛煉了供應鏈配合、製造產能擴張及超大規模集群軟件打磨的核心能力。
*   **解決客戶集中度風險：** 鑑於IPO過程中投資者對單一客戶依賴的擔憂，Cerebras大力拓展客戶基礎。
*   **OpenAI成為新的核心支柱：**

## DeepSeek-V4 预览版技术报告解读

### 一、 发布背景与核心定位

*   **发布时间：** 2024年4月24日，DeepSeek 发布并开源了 V4 预览版。
*   **当前定位：** DeepSeek 坦诚指出，V4 预览版模型发展轨迹仍滞后于 GPT-5.4、Gemini-3.1-Pro 等前沿闭源模型约 3-6 个月。
*   **核心使命：** V4 系列的核心目标并非当下全面反超闭源模型，而是 **彻底重构超长上下文场景的计算效率**，为下一代大模型在测试时扩展、长程任务、在线学习等新范式打下坚实的基础设施底座。

### 二、 DeepSeek-V4 系列核心模型

DeepSeek-V4 系列包含两款采用**混合专家模型 (MoE)** 架构的模型：

1.  **DeepSeek-V4-Pro (旗舰性能)：**
    *   总参数规模：1.6 万亿 (1.6T)
    *   单 token 推理激活参数：490 亿 (49B)
2.  **DeepSeek-V4-Flash (极致高效)：**
    *   总参数规模：2840 亿 (284B)
    *   单 token 推理激活参数：130 亿 (13B)

*   **共同特性：** 两款模型均原生支持 **100 万 token 上下文**，在开源大模型领域属前所未有的突破。

### 三、 极致效率飞跃：百万上下文与低成本推理

尽管上下文长度和参数规模大幅提升，V4 系列的推理成本反而显著降低：

*   **DeepSeek-V4-Pro：** 在 100 万 token 上下文设置下，
    *   单 token 推理 FLOPs：仅为上一代 V3.2 的 **27%**。
    *   KV Cache 占用：仅为上一代 V3.2 的 **10%**。
*   **DeepSeek-V4-Flash：**
    *   单 token 推理 FLOPs：仅为 V3.2 的 **10%**。
    *   KV Cache 占用：仅为 V3.2 的 **7%**。

**核心价值：** 上下文长度扩展近 8 倍，但单 token 算力需求大幅下降。这意味着超长上下文场景不仅可行，且成本低廉，可大规模商用。

### 四、 三大核心模块同步革新：全维度效率优化

DeepSeek-V4 首次将“稀疏化”理念拓展至 Transformer 架构的注意力机制、残差连接和优化器三大核心模块，实现从参数稀疏到上下文稀疏的全维度效率优化。

1.  **混合注意力架构 (Mixed Attention Architecture)：**
    *   **背景：** 传统自注意力机制计算复杂度随上下文长度平方级增长。
    *   **创新：** 设计全新混合注意力架构，交替使用压缩稀疏注意力 (CSA) 和重压缩注意力 (HCA)，并搭配滑动窗口分支，实现“粗粒度+细粒度，稀疏+稠密”组合拳。
        *   **压缩稀疏注意力 (CSA)：** 先将每 4 个 token 的 KV 缓存合并为摘要，再通过闪电索引器，让每个查询 (query) 从压缩摘要中挑选最相关的 top-k 进行计算。
        *   **重压缩注意力 (HCA)：** 将每 128 个 token 合并为摘要，对所有压缩摘要进行稠密注意力计算，保证长程依赖。
        *   **滑动窗口注意力分支：** 专门处理近距离 token 细节依赖。

2.  **流形约束超连接 (mHC)：**
    *   **背景：** 传统残差连接在深层网络中易出现数值不稳定问题。
    *   **创新：** 将残差映射矩阵约束到双随机矩阵流形上，确保映射矩阵的谱范数不超过 1，从而保证信号传递的非扩张性，提升深层网络训练稳定性。
    *   **额外优势：** 解耦残差流宽度与模型隐藏层维度，以极小计算开销提供新的模型扩展维度，提升表达能力。

3.  **Muon 优化器：**
    *   **背景：** 传统 AdamW 优化器在大规模训练中收敛速度和稳定性有局限。
    *   **创新：** V4 系列在除嵌入层、预测头、归一化层外的核心模块中，用 Muon 优化器替代 AdamW。Muon 优化器具有更快的收敛速度和更强的训练稳定性。
    *   **DeepSeek 优化：** 针对性设计混合牛顿-舒尔茨迭代算法，分两阶段完成正交化（前 8 步快速收敛，后 2 步精准稳定），提升效率和数值稳定性。

### 五、 全栈工程优化

为支撑新架构的落地，DeepSeek-V4 进行了全面的工程优化：

1.  **专家并行 (EP) 细粒度优化：**
    *   将专家拆分成多个波次，形成计算、通信、内存访问完全重叠的流水线内核，将跨节点通信延迟完全隐藏在计算过程中。
    *   在英伟达 GPU 和华为昇腾 NPU 平台均验证，通用推理场景提速 1.5-1.73 倍，强化学习采样场景最高提速 1.96 倍。
2.  **TileLang 内核开发与批量不变确定性内核库：**
    *   采用领域专用语言 (DSL) TileLang 快速开发融合内核，替代原生算子，实现极致运行时性能。
    *   开发端到端、按位一致的批量不变确定性内核库，解决传统算子在不同批次、硬件下输出不一致问题，确保训练稳定性和可复现性。
3.  **FP4 量化感知训练及框架优化：**
    *   MoE 专家权重和 CSA 索引器 QK 路径采用 FP4 (MXFP4) 量化，并设计无损 FP4 到 FP8 反量化流程，复用现有 FP8 混合精度训练框架。
    *   训练框架适配 Muon 优化器、mHC 结构和混合注意力机制，设计两阶段上下文并行策略，解决超长上下文训练难题。
    *   推理框架专门为混合注意力设计异构 KV 缓存结构和磁盘 KV 缓存存储方案，解决百万上下文场景下共享前缀缓存复用问题，降低高频调用成本。

### 六、 训练范式革新："分化再统一"后训练

为解决 V3.2 混合强化学习中不同能力互相干扰的问题，V4 采用两步走策略：

1.  **领域专家专项训练：**
    *   针对数学、代码、Agent、指令跟随等核心领域，分别训练独立专家模型。
    *   每个专家模型先用高质量数据监督微调，再用 GRPO 算法针对领域专属奖励模型强化学习。
    *   根据推理强度训练不同配置专家模型，形成 **Non-think、Think High、Think Max** 三档推理模式，用户可灵活选择。
        *   **Think Max 模式：** 注入强指令，拉满上下文长度和推理深度，释放模型能力上限，适合科研、复杂推理场景。
2.  **On-Policy 蒸馏 (OPD) 整合专家：**
    *   将十多个不同领域的专家模型融合成一个统一模型。
    *   让最终统一模型作为学生，自主生成回答后，匹配最擅长该问题的专家模型输出分布，在 logit 层级完成对齐。
    *   **工程方案：** 将所有教师模型权重卸载到分布式存储，仅缓存每个教师模型最后一层隐藏状态，训练时按教师索引对样本排序，确保 GPU 显存中每次只驻留一个教师模型的预测头。

#### Agent 能力专项优化

*   在后训练阶段将 Agent 作为独立专家方向进行专项训练。
*   工具调用格式从 JSON 替换为带特殊 token 的 XML 格式，降低转义错误，提升稳定性。
*   优化推理轨迹管理机制，在工具调用场景下，跨轮次推理轨迹完整保留，避免长程 Agent 任务“失忆”。
*   自研 DSec 沙箱平台 (Rust 语言开发)，单集群并发管理数十万个沙箱实例，为 Agent 强化学习训练和评测提供安全、高效、隔离的运行环境。
