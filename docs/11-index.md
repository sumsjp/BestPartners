## 红杉资本AI峰会主题演讲精华整理：AI计算革命、MAD原则与人类价值的回归

大家好，这里是最佳拍档大飞。今天为大家梳理红杉资本AI峰会主题演讲的精华内容。此次Keynote由红杉资本三位核心合伙人帕特·格雷迪（Pat Grady）、索尼娅·黄（Sonya Huang）和康斯坦丁·布勒（Konstantine Buhler）共同发表。

他们将AI定义为一场**计算革命**，而非过去的通信革命，并为AI应用创业者提出了**MAD原则**。他们明确指出**长周期智能体已到来，服务正成为新的软件**，并大胆预测**未来99.9%的认知工作将由机器代劳**。

接下来，我们将详细梳理演讲的各项核心内容。

### **第一部分：AI革命的本质与独特性（主讲人：帕特·格雷迪）**

帕特·格雷迪首先回顾了科技行业的演进路径，强调了AI的爆发是过去几十年算力、带宽、数据和人才等基础底座一步步积累的黄金节点。他进一步指出，AI这波浪潮与以往所有浪潮有本质区别：

1.  **规模最大：**
    *   AI是第一个同时覆盖**软件与服务**的技术浪潮。
    *   服务业市场空间巨大，远超软件（例如，仅美国法律服务市场就达4000亿美元，相当于整个软件产业）。潜在规模可能达到5万亿、10万亿甚至50万亿美元。

2.  **速度最快：**
    *   技术迭代、产品落地、公司成长速度呈指数级加快。
    *   未来将有更多巨头以更快的速度涌现，市场空白正以惊人速度被填补。

3.  **本质不同：计算革命而非通信革命。**
    *   过去互联网、云计算、移动通信等主要解决信息如何**分发、流转、触达**（通信革命）。
    *   人工智能解决的是信息如何被**处理**，它改变的是“思考”本身（计算革命）。这意味着技术底座每天都在刷新，赖以构建产品的基础能力随时可能被更新。

**AI发展的三次关键拐点：**
*   **2022年11月ChatGPT时刻：** 首次展现预训练模型的强大力量。
*   **o1模型：** 带来了强大的推理能力。
*   **近期Claude Code、Claude Opus 4.5等：** 展现了长周期智能体的无穷潜力。
这些跃变使得AI在功能和实践上已非常接近**通用人工智能（AGI）**。红杉认为，当智能体能从失败中自我恢复、坚持不懈完成任务时，这在商业意义上就是AGI的落地。

**AI时代的变革：**
*   **不是“更快的马”，而是“汽车”。** 许多AI应用不再是简单提升效率10%、20%，而是带来10倍、40倍的效率提升，彻底改变工作方式、工作性质和组织架构。AI的“汽车时代”真的来了。

### **第二部分：AI应用创业者的MAD原则（主讲人：帕特·格雷迪）**

在技术底座快速演进的时代，模型之上的应用公司如何构建壁垒？红杉提出了MAD原则：

1.  **M - Moats（护城河）：从客户倒推（Customer-back）而非技术外推（Tech-out）。**
    *   许多创业者本能地追求底层技术优势，但计算革命中技术能力演进极快，今天的优势明天可能被抹平。
    *   真正的持久护城河来自**客户**：与客户的深度绑定、对行业痛点的理解、围绕客户打造的全流程服务、积累的场景数据和不可替代的交付能力。这些不会因大模型更新而消失。
    *   红杉创始人唐·瓦伦丁的著名质问是“那又怎样？（So what?）”，即你的技术对客户到底意味着什么，解决了什么真问题，带来了什么不可替代的价值。

2.  **A - Affordance（示能）：极简使用路径，用户闭眼可用。**
    *   “示能”指一个东西本身就能告诉用户它能做什么、如何使用，使用路径极简，阻力几乎为零（如锤子）。
    *   许多底层模型能力虽强，但若无良好“示能”，普通用户无从下手。
    *   应用层创业者的机会在于，为特定用户和场景打造阻力最小的路径，让用户直接获得业务结果，而非学习复杂操作。

3.  **D - Diffusion（扩散鸿沟）：基础模型演进与用户采纳速度的落差。**
    *   基础模型的演进速度远远快于企业、市场和普通用户的采用速度，这个落差每天都在拉大。
    *   这个巨大的落差，就是应用层创业者的机会：基础模型实验室负责技术突破，而应用层创业者负责将技术**填平到场景、产业和每一个用户的工作流中**。扩散鸿沟越大，应用层的价值就越大。

**MAD原则总结：** 用客户倒推构建护城河，用极简路径实现示能，用扩散鸿沟抓住机会。

### **第三部分：智能体（Agents）的崛起与“服务即软件”（主讲人：索尼娅·黄）**

2022年早期智能体（如AutoGPT）虽曾爆红，但因模型未成熟而缺乏实用价值。但红杉判断，到**2026年，智能体已不再是玩具，而是能真正带来商业价值的工具**，Claude Code和OpenClaw等产品是其标志性验证。

**智能体的定义与核心功能：**
智能体是一个能够感知环境、选择行动，并自主朝着目标推进的系统，具备以下核心功能组件：
1.  **推理和规划能力：** 智能体的基础直觉，能临场思考、拆解任务、制定步骤。
2.  **采取行动的能力：** 调用工具（搜索、编码、操作软件、访问系统等），人类过去为自己打造的工具几乎都可被智能体复用。
3.  **向目标持续迭代的能力：** 智能体能长时间试错、纠错、调整和恢复，直至完成任务，实现**持久性**。
因此，智能体的本质是**“把事情搞定的能力”**。

**智能体的构成与突破：**
智能体由模型（大脑）、工具（手脚）和支撑框架（Harnesses，提供持久性）三部分组成，过去一年都取得了飞速突破：
*   **模型：** 处理复杂任务、持续不跑偏的时间从几十分钟提升到几小时，足以支撑长周期任务。
*   **工具：** 人类已有的海量工具直接成为智能体的能力外延。
*   **支撑框架：** 提供持久性，结合强化学习，智能体在各类场景中不断提升性能，甚至出现**“机器构建机器”**的初步自我进化迹象。

**智能体自主性光谱的演进：**
智能体正从辅助人类（如Tab键自动补全）向自主开发、管理智能体团队发展。未来将出现后台智能体、异步智能体，甚至能生成子智能体的智能体，最终达到“黑灯工厂”模式——**智能体自主完成全流程任务，无需人类审核**（如网络安全领域已实现）。智能体将从助手进化为可靠员工。

**终极判断：服务即新的软件（Service as the new Software）。**
*   智能体拥有人类难以匹敌的优势：易规模化、无需情绪维护、成本低廉（仅需Token成本）。
*   智能体的部署速度将史无前例，因为经济账清晰且具有比特的可扩展性。
*   当智能体能进行商业交易、互相支付和谈判，并在后台巡视安全时，世界将发生巨大变化。
*   红杉预测，**曾经以为需要100年才能实现的未来，依靠智能体，可能100天内就会实现。**

---

### **第四部分：认知工作的范式转移与人类价值的终极回归（主讲人：康斯坦丁·布勒）**

康斯坦丁·布勒将工作分为物理劳动和认知劳动，并指出认知领域将重演物理劳动的历史：

*   **物理劳动：** 工业革命使机器接管了99%以上的物理劳动（搬运、运输、制造等），构成了现代文明的基础。
*   **认知劳动：** 思考、推理、规划、创造、设计、分析。在人类历史绝大多数时间里由人类完成。**未来，AI将使其重演物理劳动的历史，地球上99.9%的认知工作将由机器代劳。** 这是一场规模更庞大、速度更迅猛、影响更深远的认知革命。

**理解未来：四个极具穿透力的故事**

1.  **铝的隐喻：** 19世纪中叶，铝是珍贵金属；但电解法发明后，铝变得普惠且廉价，用于糖果锡纸。
    *   **启示：** 曾经稀缺、需要几十年修炼的“博士级珍贵技能”，未来会像铝一样，变成可以瞬间调用、随手可得的**普惠基础设施**。

2.  **异类设计（Alien Design）：** 今天的世界一切为人类设计，优化逻辑符合人类直觉。
    *   **启示：** 当机器接管认知工作后，设计逻辑会完全不同。AI设计的产品（如NASA优化的卫星天线）可能不符合人类审美和几何直觉，但性能远超传统。我们必须接受一个不再完全以人类思维为中心的世界。

3.  **新兴科学的诞生：** 工业革命早期，工程师摸索改进蒸汽机百年，直到卡诺建立热力学才系统化。
    *   **启示：** 今天的AI仍处摸索修补阶段。未来几十年，一定会诞生一门像热力学一样基础、重要的**“智能科学”**，帮助人类真正驾驭AI，甚至参透意识本质。

4.  **非理性的艺术：** 摄影技术出现，写实绘画被机器替代，艺术转而追求印象派、表现主义，转向表达内心、情感和灵魂。
    *   **启示：** AI时代，艺术会再次重构。人类不再比拼写实、技巧或记忆，而是回归到最本质的**情感体验、内心表达和灵魂连接**。技术越强大，人类越要回到感性的内核。

**当AI接管99.9%认知工作，人类的价值何在？**

*   **人是万物的尺度（普罗泰戈拉）：** 脱离人的体验、情感、连接和价值判断，任何事物都无意义。
*   **不可替代的核心价值：** AI可以代劳工作，提供效率，产出结果；但人与人之间的**深度连接、情感交互、信任、热爱、价值定义**，才是技术洪流中最持久、最不可替代、最核心的价值。
*   **总结：** 铝会贬值，技能会贬值，知识会贬值；但人的感受、人的连接、人的情感，永远不会贬值。**AI让效率极致化，而人类让意义永恒化。**

---

### **总结与展望**

红杉的这场演讲不仅描绘了AI的宏大图景，更指明了我们应如何面对和构建未来，以及如何在技术洪流中坚守人的价值。

*   **对创业者：** 不要盲目追逐技术，要回归客户、场景和价值。
*   **对职场人：** 不要害怕被替代，要转向情感、连接、创造和意义等机器无法复制的领域。
*   **对普通人：** 未来可能不是机器统治人类，而是让机器来承担劳作，而人类回归生活本身。

如果“100年的愿景真的能在100天内实现”，那不仅是技术魔幻，更是因为我们找到了正确的方向：**用技术解放劳动，用人类定义价值。**

## 洪乐潼：一位24岁的AI数学新星，引领Axiom颠覆数学证明领域

### 引言

「最佳拍档」主持人大飞介紹了一位年僅24歲的中國籍「00後」女性——洪乐潼，她正帶領一支由全球頂尖科學家組成的團隊，利用AI實現嚴謹的數學定理自動證明，並成功創立了估值高達16億美元的明星公司Axiom。更令人驚訝的是，一位57歲的美國終身數學教授毅然放棄「鐵飯碗」，主動加入她的團隊。這一切源於張小珺在硅谷對洪乐潼進行的一場深度訪談，本文將分享其成長、創業故事、AI與數學結合的突破，以及硅谷創業的真實內幕。

### 洪乐潼：不凡的成長之路

1.  **多元文化薰陶與數學啟蒙：**
    *   廣州長大，心懷家鄉的煙火氣和美食。
    *   童年受《百家講壇》易中天講三國、諸子百家及黃永玉作品的影響。
    *   母親是北京人，雙重文化背景塑造了她。
    *   因母親數學不好，她從小被鼓勵接觸數學，從此深陷其中。

2.  **不懈的「笨鳥先飛」精神：**
    *   自認從小到大都是環境中最「笨、不起眼」的人，而非天才型。
    *   奧數班曾從4班努力擠進24班，面對歐式幾何難題時，她不是靠直覺，而是用「蠻力」——將幾何元素轉化為符號表達，純靠計算硬推，哪怕花費數倍時間。這種「大力出奇蹟」的思路，與後來AI的工作邏輯不謀而合。

3.  **從奧賽競爭到高等數學探索：**
    *   高中厭倦奧賽的「零和遊戲」，轉而鑽研高等數學（微積分、實分析、複分析）。
    *   她認為高等數學是「正和遊戲」，鼓勵探索、創造和討論，而非輸贏。這在她心中埋下了數學是探索而非競爭的種子。

4.  **MIT的洗禮與堅韌：**
    *   受電影《心靈捕手》影響，高中畢業毅然申請MIT，主修數學與物理雙學位。
    *   在MIT，她再次感到自己「最笨」，身邊都是IMO金牌得主。大一選博士級概率論課僅考4分（滿分40，平均9分）。
    *   但她從未放棄，自稱「打不死的小強」，一道題三個月、六個月不放棄思考，這種韌性是她創業成功的核心。
    *   大學期間獲北美數學本科生最高榮譽「摩根獎」，但她謙遜地認為這是隨機性。
    *   研究數論中的模形式和橢圓曲線時，體會到數學簡潔又深邃的美，徹底「淪陷」。

5.  **跨界探索與創業火花：**
    *   本科後赴牛津攻讀神經科學碩士（因親人健康問題），但因無法接受動物實驗轉向計算神經科學，該領域與AI緊密相連，激發了她對AI的熱情。
    *   碩士後又入斯坦福，同時攻讀數學博士和法學博士。法學院對美國憲法文本主義的嚴謹解讀，讓她聯想到數學定理證明，突然萌生了「既然AI能分析法律文本，為何不能做數學證明？」的閃電般念頭，最終下定決心投身「AI for Math」。

### Axiom：AI for Math領域的崛起

1.  **夢幻合夥人：**
    *   在硅谷一家咖啡館，她與Meta AI研究總監、CUDA第一批開發者舒布霍·森古普塔相識。
    *   兩人長聊一年半，從未暴露真實身份，卻發現彼此理念高度契合：AI for Math需工業級算力與工程能力才能落地。

2.  **痛苦的創業抉擇：**
    *   2024年秋，洪乐潼正式決定輟學創業，但內心極度掙扎，曾花兩個月自我勸退。她自小嚮往學術，覺得創業是「不務正業」。
    *   最終讓她下定決心的是，她通讀了數百篇AI for Math論文，發現這是一個「工程問題」而非純研究問題，科技風險可控。且因競品公司只招計算機博士而拒絕了她，更堅定了她「自己幹」的決心。

3.  **融資奇蹟與誠信策略：**
    *   她厭惡枯燥的融資過程，形容其為「復讀機」。
    *   但她遇到了真正懂她的投資人，如B Capital聯合創始人、身兼數學家的霍華德·摩根。
    *   Axiom種子輪融資6400萬美元，估值3億美元；半年後A輪融資超2億美元，估值飆升至16億美元，成為硅谷獨角獸。
    *   作為24歲的華人女性，在深科技領域融資曾面臨挑戰，但她堅持「極致坦誠」，不誇大前景，不隱瞞風險，反而打動了投資人。

4.  **夢之隊的組建：**
    *   初期策略：前15名員工不招數學家，先建AI與工程技術底座。
    *   最終團隊匯集了AI、代碼生成、數學三大跨界精英：Meta AI強化學習專家、Yann LeCun團隊核心代碼生成工程師、5位IMO國際數學奧賽金牌得主，以及頂級數論學家和形式化語言專家，幾乎囊括了AI for Math領域頂級論文的作者。
    *   **標誌性人物：** 57歲的美國終身數學教授小野肯（Ken Ono），繼承拉馬努金精神的數論大師，前美國數學學會副主席，放棄OpenAI邀約，毅然加入Axiom，成為團隊的精神領袖。他選擇Axiom的原因是團隊更懂數學、氛圍更純粹，專注於數學本身而非AGI的附屬品。

5.  **服務型領導力：**
    *   面對團隊中資深科學家，洪乐潼自認不是「命令型CEO」，而是「服務型領導者」——像爬山時在後方遞水、掃清障礙、保障後勤的人。
    *   Axiom延續Meta的「自下而上」文化，無辦公室政治、人事糾紛，所有人專注技術與研究，這是吸引頂級人才的關鍵。

### 核心技術與商業價值

1.  **AI for Math 的真諦：**
    *   不是AI解數學題，而是**自動定理證明（ATP）**。計算機科學家已研究數十年，旨在用規則系統驗證數學證明。
    *   當證明複雜時，需要數學家與機器合作，用專用語言實現**交互式定理證明（ITP）**。
    *   Axiom的突破：用AI取代ITP中的人類，實現AI驅動的嚴謹數學證明。

2.  **核心工具與理論：**
    *   **Lean：** 一種形式化證明語言，將數學證明轉化為可運行、可驗證的代碼，確保每一步邏輯和推導的絕對正確性。
    *   **庫里-霍華德同構（Curry-Howard Correspondence）：** 強調數學證明與計算機程序之間的一一對應關係，Axiom的AI因此既能做數學證明，也能做代碼驗證。

3.  **里程碑式的成就：**
    *   Axiom的核心產品**AxiomProver**，在普特南大學生數學競賽中取得滿分。這是98年來第六個滿分得主，也是第一個AI滿分得主。
    *   AI的解題思路與人類截然不同：人類靠幾何直覺和創造力，AI則靠「枚舉、分類討論」的「蠻力硬推」，印證了洪乐潼「AI是最完美的蠻力型數學家」的觀點。
    *   她認為，人類數學家提出猜想、提供直覺，AI負責驗證、推理，形成完美互補。

4.  **技術路徑的選擇：**
    *   Axiom與谷歌DeepMind的AlphaProof路徑不同。AlphaProof採用蒙特卡洛樹搜索，算力成本高昂。
    *   Axiom則借鑒字節跳動豆包Seed-Prover的思路，更注重推理效率和工程落地，避開高成本搜索，並自研12-13個輔助工具優化Lean語言執行速度，解決形式化證明效率低、數據少的痛點。

5.  **千億級的商業市場：**
    *   洪乐潼明確指出，**驗證**是Axiom的第一個核心市場，價值千億美元。
    *   **芯片驗證：** 現有工具效率低下。亞馬遜曾花3-5年寫26萬行代碼驗證一個CPU組件，Axiom的AI可將時間壓縮到幾天甚至幾小時，成本降低數十倍。
    *   **代碼驗證與分布式系統驗證：** Axiom的AI能直接驗證代碼邏輯，無需大量測試用例，實現100%正確代碼，對互聯網、金融、航天等領域具顛覆性意義。

### 洪乐潼的願景與哲學

1.  **AI的終極形態：超級智能（ASI）而非通用智能（AGI）：**
    *   她不相信通用智能（AGI），堅信專業化的**超級智能（ASI）**才是未來。
    *   人類智能本身就是專業化的，因此打造數學領域的專屬超級智能，再泛化至代碼、物理、芯片等領域，形成「專屬價值扇形」，而非追求全知全能的AI。
    *   她預言AI for Math將讓人類社會從「數學匱乏」走向「數學豐富」，AI將解決所有未解理論數學問題，應用科學家的數學瓶頸將被輕鬆突破。人類數學家則轉變為直覺提供者、資源分配者和猜想提出者。

2.  **「對苦難上癮」與領導力：**
    *   她認為，幾乎所有成功的創業者都「對痛苦和折磨上癮」，這也是投資人青睞的特質。
    *   自己的成長歷程充滿失敗，但正是這些失敗塑造了她的韌性，讓她成為「打不死的小強」。失敗是常態，是成長的必經之路。
    *   她對領導力的理解是「服務」，CEO是為團隊掃清障礙、提供資源和環境的人。

3.  **對中國AI的評價與全球協作：**
    *   高度讚揚字節跳動的豆包Seed-Prover，認為其在AI定理證明領域表現出色，執行力強，技術思路與Axiom高度契合。
    *   她認為中美AI各有優勢，人才不分國界，未來科技創新需要全球學術信任與純真，少一些商業顧忌，多一些科研初心。

### 對2026年AI發展的三大預測

1.  小型創業公司和新型實驗室將推出第一個真正落地的**持續學習模型**，使AI能像人類一樣不斷學習新知而不遺忘舊知。
2.  **多模態大模型**將迎來新突破，實現對多模態信息的真正邏輯理解。
3.  **形式化驗證工具**將成為強化學習的核心獎勵信號，被大規模應用，徹底解決AI的決策不確定性。
    *   她最核心的賭注是「系統而非模型」，堅信**遞歸自我提升**將很快實現，AI能自己優化自己，傳統SaaS模式將被顛覆。

### 結語：永遠的學徒

*   洪乐潼推薦的讀物：《初等數論》、《分析數論》、《紅樓夢》、《大雅寶胡同甲2號》，這些書給予她力量。
*   她認為訓練AI與訓練自己如出一轍：循序漸進，不斷挑戰極限。
*   她人生最快樂的時光是通讀AI for Math論文的日子，如今那些論文的作者大多已成為Axiom的戰友。
*   她不在乎自己的墓志銘，只希望永遠是個學徒，在數學、AI、科技的世界裡永遠保持好奇、學習和探索。

這個24歲的華人姑娘，以其熱愛、堅持和韌性，在科技最前沿的AI for Math賽道上站穩腳跟，並引領著一群頂級科學家，推動著人類數學與AI的共同進步。

### **Paul Tudor Jones 深度播客訪談精要：善意、交易哲學、AI風險與市場預警**

這份訪談來自華爾街傳奇宏觀交易員 Paul Tudor Jones（PTJ），他因在1987年股災中精准做空而聲名鵲起，是管理數百億資金的頂級風險管理者，更是一位將交易、慈善與人生哲學融為一體的智者。他用50年真金白銀的經驗，分享了極具價值的干貨，包括公開向巴菲特道歉、揭露AI行業最驚悚的真相，以及對美股的明確預警。

#### **一、善意的漣漪效應：比任何交易技巧都重要**

Paul Tudor Jones 將“善意”視為訪談中唯一希望聽眾記住的內容，重要性超越任何交易技巧和市場判斷。

1.  **童年記憶的種子 (1957年)：**
    *   他兩歲半時在生鮮市場與母親走散，一位年長的黑人先生溫柔地牽著他的手，四處尋找並安慰他。
    *   找到母親後，這位先生堅決拒絕了母親5美元的酬謝，只說：「不用謝，夫人。我知道換作是你，你也會為我的孩子這麼做。」
    *   這件小事成為PTJ人生中第一個也是最深刻的記憶，他在接下來的10-12年裡，每天為這位無名先生祈禱，深信這份善意在他心中埋下了一顆種子。

2.  **“我有一個夢想”：善意的傳遞與放大 (1986年)：**
    *   32歲時，PTJ觀看節目，被商人尤金·朗承諾資助哈萊姆區貧困孩子上大學的故事（“我有一個夢想”項目）深深打動。
    *   他將此視為童年經歷的“反向復刻”——當年是黑人先生幫助白人男孩，如今是商人幫助貧困的黑人/拉丁裔孩子。
    *   他決定效仿，選擇紐約犯罪率最高的貝德福德-斯泰弗森特（Bed-Stuy）社區，承諾資助孩子們上大學。

3.  **從熱情到專業：慈善的進化與成果：**
    *   初期只給錢和承諾並不足夠，三年後孩子們面臨學業困難、暴力、輟學、未婚懷孕等問題，甚至有孩子離世。
    *   PTJ意識到，對抗貧困需有**計畫、方法和專業體系**。
    *   他聘請專業家教，完善教育方案，並在1987年股災後創辦了著名的**羅賓漢基金會（Robin Hood Foundation）**，用商業邏輯和量化標準做慈善。
    *   90年代末，他在該社區創辦了**貝德福德-斯泰弗森特卓越特許學校（Bed-Stuy Charter School of Excellence）**，幾年內成為紐約市小學第一名。
    *   PTJ強調，這一切都源於當年陌生先生的一次舉手之勞，那份小小的善意經過幾十年的傳遞和放大，最終改變了成千上萬人的命運。

4.  **善意有漣漪效應：**
    *   他鼓勵每個人每天刻意做一件小小的善事，相信世界會因此發生巨大變化。
    *   他指出2000年前美國社會並無現在如此多的惡意攻擊和對立，呼籲年輕人不要將當前的戾氣視為常態，而是選擇用善良和尊重重建社會信任。

---

#### **二、交易員與投資者的本質區別：向巴菲特致敬**

Paul Tudor Jones 坦言，他這輩子最大的感受是羨慕投資者，尤其是沃倫·巴菲特，甚至希望自己能重來選擇成為一名投資者，因為生活會輕鬆太多。

1.  **“橄欖球場戰壕裡的右護鋒”：**
    *   PTJ將自己比作在橄欖球場戰壕裡拼搏了50年的右護鋒，每一天都在廝殺、對抗。
    *   他的入行經歷（1976年通脹失控，商品交易員）讓他體驗了超高波動率市場的極端風險。

2.  **“白銀之王”的史詩級崩盤：流動性至上：**
    *   他講述了邦克·亨特在70年代末瘋狂囤積白銀，從世界第二富豪一度成為世界首富，最終因COMEX交易限制導致白銀崩盤，六週內幾乎破產的案例。
    *   這一幕給PTJ留下深刻印記，加上祖父“真正的財富是你明天能開出支票的金額”的教誨，讓他**一生將流動性放在第一位**。
    *   在超高波動率環境中，長期持有和價值投資曾顯得天方夜譚。

3.  **對巴菲特的誤解與公開道歉：**
    *   很長一段時間，PTJ年年吐槽巴菲特，認為他只是運氣好趕上美國大牛市，並非投資天才，若在日本投資日經指數絕不會成功。他沉浸於短線交易，無視復利力量。
    *   直到他聽了一期關於伯克希爾·哈撒韋的播客，才震驚地發現巴菲特9歲就理解復利，17歲拜師格雷厄姆，並與查理·芒格結合，成就了投資傳奇。
    *   PTJ在播客中**非常真誠、鄭重地向巴菲特公開道歉**，承認自己一生完美避開了復利這一最強大的財富工具。

4.  **兩種截然不同的路：**
    *   **PTJ（交易員）：** 管理的BBI基金與標普500相關性為-0.12，所有收益均為超額收益（Alpha），不依賴大盤漲跌。他是一個風險管理者，每天在市場廝殺，靠判斷、執行和風控生存。
    *   **巴菲特（投資者）：** 基於對美國的堅定信仰，長期持有、穿越波動，靠信仰、復利和趨勢賺錢。
    *   PTJ坦言羨慕巴菲特的信仰體系和心態（在2008-09年資產回撤50%仍淡定從容），承認自己缺乏那種耐心、堅韌和定力。

#### **三、AI：最可怕的“零風險管理”狀態**

Paul Tudor Jones 認為，任何成功的交易員和投資者首先是頂級的風險管理者，而現在的AI行業，是他見過最離譜和最危險的“零風險管理”狀態。

1.  **令人毛骨悚然的行業共識：**
    *   約18個月前，PTJ參加了一場有全球四大頂級AI模型公司研究員參與的閉門會議。
    *   他直接提問AI安全問題如何解決，得到的答案讓他**毛骨悚然**：這些頂尖專家認為，**等5000萬到1億人因AI事故死亡後，人類才會真正採取行動去監管。** PTJ用“瘋狂”來形容這個答案。

2.  **AI與原子彈：試錯成本的巨大差異：**
    *   人類創新通常是“構建-打破-迭代”模式，但AI的試錯成本是數以億計的生命。
    *   原子彈發明後，美國國會在18個月內就成立原子能委員會進行嚴格監管。而AI發展三年多，全球幾乎沒有有效監管，完全野蠻生長。

3.  **AI的三大核心風險：**
    *   **安全風險不可逆轉：** 一旦AI系統打破，就是災難性的，可能危及數億人生命，沒有回頭路。
    *   **缺乏公眾話語權：** AI的發展速度、技術方向由少數科技公司決定，沒有公眾監督和否決權。
    *   **社會衝擊被嚴重低估：** AI將對勞動力市場造成顛覆性衝擊，大量崗位被替代，而社會對此毫無準備。
    *   巴菲特也同意PTJ觀點，感嘆“AI的精靈已經從瓶子裡跑出來了，不知道還能不能塞回去。”

4.  **PTJ提出的最緊急解決方案：**
    *   **強制所有AI生成內容加水印。** 這是恢復社會信任、區分真實與偽造的核心手段。他甚至建議將故意違規三次定為重罪，直接入獄。
    *   他擔憂若不強制水印，未來虛假信息將摧毀整個社會的信任體系。
    *   對於未來人類大腦植入晶片、人機融合等趨勢，PTJ明確表示堅決反對，認為必須在一切太晚之前建立規則，守住底線。

#### **四、交易智慧與市場預警：泡沫正在醞釀**

Paul Tudor Jones 作為50年的交易員，對風險的敏感度遠超常人。他強調，好的風險管理者會在危機發生前就控制風險。

1.  **導師伊萊·塔利斯：情緒捕捉與逆境心態：**
    *   PTJ的導師塔利斯擅長精準嗅探市場極端情緒，在極度貪婪時賣出，極度恐懼時買入，耐心等待“一擊致命”的機會。
    *   塔利斯在一次棉花交易巨虧後，仍能衣著得體、談笑風生，用行動教會PTJ：**當困境來臨時，內心的信念和從容才是翻盤的根本。**

2.  **交易如拳擊：觀察、等待、重拳出擊：**
    *   市場是你的對手，大部分時間在觀察、等待、累積信息，贏下小回合。
    *   偶爾會出現絕佳機會，可以打出“重拳”，直接KO對手。
    *   他舉例了三個經典“KO”案例：

*   **清華時期的啟示：** 曾組建天空工廠社團，追求技術極致，但大疆的成功讓他意識到創業必須滿足「市場真實需要」，而非僅做「酷」的東西。

*   **N+1邏輯：** 這是追覓商業體系的核心支柱，與傳統N-1模式（降低成本做平替）相反。
    *   **N-1模式：** 過去中國企業的主流，現已走到極限，將企業鎖死在追隨者位置。
    *   **N+1模式：** 繼承行業成熟知識（N），避開失敗路徑，在此基礎上，挑選幾個用戶可清晰感知、帶來體驗升級的點做創新增量（+1）。他認為這種模式商業價值是N-1的十倍，能支撐品牌高端化、全球化。
*   **創新四象限模型：** 為N+1落地而生。
    1.  **緊盯成功公司：** 學習既有知識，做微創新與增量。
    2.  **關注創新公司：** 借鑒思路或避開無效路徑。
    3.  **跨行業移植：** 將其他賽道有效方法遷移。
    4.  **回歸第一性原理：** 從用戶需求本源思考（但成功率低）。
    *   **結論：** 高效創新是深度繼承行業經驗，再做精準的+1突破，而非推倒重來。

*   **資金運用：** 反駁造車、造手機是無底洞的觀點。認為是「造錯」才費錢。造車投入100-300億人民幣，分5年，每年僅20億左右。追覓融資數十億，但燒錢極少，資金使用效率頂尖，足以支撐多賽道佈局。
*   **敬畏心：** 俞浩早在2013年就走訪塔塔工廠，試駕過百款車，對全球汽車產業鏈有深入理解，而非盲目跨界。

**【高端化與全球化戰略】SABCD品牌五層軌道理論**
追覓從小米生態鏈的「E級代工廠」成功轉型高端化，依託一套完整的品牌理論：
*   **SABCD品牌五層軌道：** 俞浩走遍全球高端商場，總結出此理論，不同層級品牌（S/A/B/C/D）對應不同利潤率（S級可達40%+）。S級品牌往往與全球最貴的300個地標位置綁定，稀缺性是品牌價值的核心。

*   **成果：** 追覓在清潔家電領域逆襲，從第四名反向操作，放棄低利潤市場，堅定走高端，實現全球利潤率第一（最近一季度淨利率20%以上，目標30%）。

**【組織管理哲學】反傳統、反官僚的「5×1000矩陣」**

    *   **時間：** 2026年3月26日上線。
    *   **Anthropic 意圖：** 優化會話緩存以降低延遲和成本。設計邏輯為：若用戶會話閒置超過一小時，重新恢復對話時，系統自動清理舊思考記錄，只保留必要上下文。
    *   **設計缺陷：** 代碼實現出現致命錯誤，清理操作未被限制為一次，而是變成了觸發閾值後「每一輪對話都執行清理」。
    *   **實際影響：**
        *   **「失憶」現象：** 一旦會話曾閒置超過一小時，之後每次提問，Claude 都會丟棄之前所有思考過程，只保留最新一輪推理記錄，導致健忘、重複操作、邏輯前後矛盾，甚至忘記最初需求。
        *   **用量異常消耗：** 因思考記錄被持續丟棄，每次請求都相當於緩存未命中，模型需重新計算，導致用戶 Token 消耗速度遠超正常。
    *   **發現難度：** Anthropic 承認花了整整兩周才定位。原因包括：必須滿足特定觸發條件（會話閒置一小時），且內部兩個不相關實驗恰好掩蓋了其復現路徑。
    *   **測試盲點：** 此缺陷通過了人工代碼審查、單元測試、端到端測試及內部狗糧測試（Dogfooding）所有環節，無一攔截。
    *   **修復：** 2026年4月10日徹底修復此 Bug。

3.  **問題三：系統提示詞（System Prompt）增加長度限制**
    *   **時間：** 2026年4月16日，伴隨 Opus 4.7 版本發布。
    *   **改動：** 在系統提示詞中加入限制規則：工具調用之間的文本不超過25個詞，最終回復不超過100個詞（除非任務確實需要更多細節）。
    *   **Anthropic 意圖：** 控制 Opus 4.7 的「啰嗦」傾向，提升閱讀效率。內部測試未發現任何性能回歸。
    *   **實際影響：** 用更全面、貼近真實場景的評估套件測試顯示，編碼質量直接下降3%。對於專業級 AI 編程工具，3% 代表能力大幅下降。長度限制壓縮了模型的規劃空間、推理步驟和解釋能力，使其無法完整展示思考過程，導致代碼質量下滑，用戶難以理解其思路。
    *   **修復：** 2026年4月20日，此系統提示被正式回滾。

**三、 事故總結與時間線**

*   **問題上線：** 3月4日（推理強度）、3月26日（緩存 Bug）、4月16日（提示詞限制）。
*   **問題修復：** 4月7日、4月10日、4月20日依次解決。
*   **持續時間：** 整個事故從3月初開始，到4月20日完全收尾，持續近一個半月。

**四、 反思：為何測試防線失效？ (Anthropic 的深刻洞察)**

Anthropic 的復盤揭示了導致問題逃逸測試的三層原因：

1.  **內部員工使用版本與公開用戶版本存在差異：** 內部測試環境、參數配置及功能開關與線上實際用戶環境不完全一致。內部員工使用的可能是「安全版」或「調試版」，導致 Bug 藏匿於細微差異中，直到大規模上線才爆發。
2.  **評估套件覆蓋面不足：** 內部標準評估集只能覆蓋常規場景和簡單任務，無法模擬用戶真實工作中的複雜情況。AI 產品的評估體系必須與產品一同持續迭代和擴展。
3.  **早期用戶反饋難以與正常波動區分：** AI 模型本身具有一定的輸出隨機性，用戶主觀感受也受多種因素影響。當問題剛出現時，零星的「變笨了、不好用了」等反饋，容易被誤認為正常性能波動，而非系統性故障，導致問題擴大後才被重視。

**結論：** 這三層原因疊加形成可怕盲區。每一道防線單獨看都很合理、完善，但組合在一起時，卻無法有效攔截那些隱蔽、場景化、非典型的缺陷。

**五、 里程碑式的發現：AI 審查 AI 代碼**

*   **回測結果：** 事故後，Anthropic 團隊使用 Opus 4.7 模型對出問題的緩存優化 PR 進行回測式代碼審查，結果顯示 Opus 4.7 能夠準確發現該 Bug，而上一代 Opus 4.6 則做不到。
*   **重要意義：**
    1.  **模型能力實質性提升：** 新一代模型在理解複雜系統交互、邏輯分支、邊界條件、狀態流轉方面有顯著進步。
    2.  **AI 輔助 Code Review 成為可能：** 用更強的 AI 模型審查 AI 產品自身的代碼，正從概念轉變為真實可用的工程實踐。Anthropic 已內部全面推行此模式，並計劃將優化後的審查工具開放給用戶。

**六、 Anthropic 的改進措施**

為應對此次事故，Anthropic 提出一整套改進措施：

1.  **統一版本：** 強制統一內部測試版本與公開線上版本，消除環境差異盲區。
2.  **強化評估：** 每次系統提示詞變更都必須針對每個模型跑全面、完整的評估，最大限度避免隱性質量下滑。
3.  **灰度發布：** 建立浸泡期與灰度發布機制，新功能先小範圍放量、充分觀察、確認穩定後再逐步擴大。
4.  **升級工具：** 升級 Code Review 工具，增強上下文理解能力，支持引入更多代碼倉庫作為背景，結合 AI 能力提升審查覆蓋率與精準度。
5.  **透明溝通：** 開通 @ClaudeDevs 社交賬號，主動解釋產品決策邏輯、參數調整原因及功能目標，增強用戶信任。
6.  **用戶補償：** 重置所有訂閱用戶的用量限額，彌補事故期間的異常消耗。

**七、 事故的幾個啟示**

1.  **使用者需建立質量基線感知：** 隨著我們越來越依賴 AI 完成關鍵工作，AI 產品的質量波動直接影響產出與判斷。用戶應清楚了解工具的正常水平，一旦偏離基線能快速察覺，及時調整或等待修復，避免將工具問題誤認為自身問題。
2.  **產品工程質量與模型能力同等重要：** 本次事故中，Claude 底層模型未變，但因默認參數、緩存邏輯、提示詞這三個產品層面的改動，用戶感知到的智能水平劇烈波動。模型再強大，若產品工程拉胯，用戶體驗仍可能崩潰。
3.  **小改動可能引發大事故：** AI 產品的發布管控必須極度謹慎。三個問題均非顛覆性重構，僅是局部改動，但疊加後造成大規模體驗惡化。AI 系統耦合度極高，任何微調都可能在複雜場景中觸發連鎖反應，研發團隊必須敬畏複雜度，嚴格控制發布流程。

**八、 結語**

Claude Code 質量事故表面看是三個 Bug 導致的體驗下滑，本質上是 AI 產品研發、測試、發布全流程的典型案例。它警示我們，AI 時代的產品工程面臨著與傳統軟件完全不同的挑戰：更高的複雜度、更強的用戶敏感度、更隱蔽的缺陷，以及更難感知的質量波動。未來，AI 將深入更多行業和崗位，類似的產品質量事故大概率仍會出現，因此我們必須始終保持對工程的敬畏之心。

**AI時代的范式革命：安德烈·卡帕西的深度洞察**

**引言：頂級工程師的“落後感”與范式躍遷**

在2026年4月29日紅杉資本的AI Ascent 2026大會上，OpenAI前聯合創始人、特斯拉前AI總監安德烈·卡帕西（Andrej Karpathy）分享了他過去一年深切感受到的技術迭代衝擊。這位擁有十幾年編程經驗的頂級程序員直言自己從未如此“落後”，並深入剖析了從軟件1.0到3.0的范式躍遷、智能體工程的興起、AI能力的“鋸齒化”缺陷，以及人類在AI時代不可替代的真正價值。這一切都指向一個結論：我們正經歷的不是漸進式優化，而是一場徹底的范式革命。

**一、卡帕西的真實體驗：從輔助工具到“氛圍編程”**

卡帕西以自身經歷開場。他從智能體工具Cursor發布以來便持續使用。

*   **初期體驗：** 最早，這些工具僅能生成簡單、偶爾有邏輯錯誤的代碼片段，雖能提升效率但能力有明顯天花板，他只將其視為輔助工具。
*   **轉折點（2025年12月）：** 在最新一代大模型的加持下，智能體生成的代碼質量已近乎完美，能穩定可靠地滿足複雜需求，他已完全記不清上次手動修改模型輸出的時間。
*   **結果：** 他開始毫無保留地信任這套系統，並正式開啟了“氛圍編程”（Vibe Coding）模式。他強調，這標誌著AI不再是單一的工具輔助，而是基於智能體的、連貫完整的工作流，真正實現了顛覆性的落地。這種認知刷新，讓他徹底沉迷其中，並快速實現了無數個業餘項目。

**二、計算范式的演進：從軟件1.0到3.0**

卡帕西隨後深入闡述了從軟件1.0到3.0的三代范式躍遷，這是理解未來軟件開發和計算架構的關鍵。他明確提出，大語言模型（LLM）本質上是一種“全新的計算機”，是全新的計算範式。

1.  **軟件1.0時代：**
    *   **核心：** 人類手工編寫顯式的規則代碼（C++、Java、Python）。
    *   **本質：** 开发者將問題精確拆解為邏輯步驟，機器被動執行。编程核心是人的邏輯表達能力。
2.  **軟件2.0時代：**
    *   **核心：** 訓練神經網絡的權重。
    *   **本質：** 編程不再是手寫規則，而是創建數據集、定義目標函數、選擇神經網絡架構，讓模型在數據中自主學習（如圖像識別、語音翻譯、早期自動駕駛）。開發者重心從寫代碼轉為整理數據和模型訓練。
3.  **軟件3.0時代：**
    *   **核心：** 大語言模型作為一台“可編程的通用計算機”。
    *   **本質：** 在足夠龐大的任務集上，用整個互聯網數據訓練GPT等大模型，使其在所有領域進行多任務處理。編程的核心已徹底變為“提示詞編程”（Prompting），用戶在模型上下文窗口中輸入的內容，就是操控這台新型計算機的槓桿，大模型會解析意圖並自主執行任務。

**卡帕西的實例證明：舊範式的冗餘**

卡帕西舉例說明，軟件3.0時代將使舊範式下的許多工作和應用變得冗餘：

*   **OpenClaw安裝：** 在1.0時代需要編寫複雜的跨平台Bash/Shell腳本；在3.0時代，只需將一段文本複製給智能體，它便能自主分析環境、安裝、調試。
*   **MenuGen項目：** 旨在解決無配圖菜單問題。他最初用傳統模式開發了一個涉及前端、後端、OCR、圖像生成、部署的完整應用（1.0+2.0混合模式）。但在3.0版本中，用戶只需拍下菜單照片發給Gemini，指示其用NanoBanana覆蓋圖片，即可直接返回帶圖菜單，完全不需要獨立App。他意識到之前的所有開發工作都是多餘的。

卡帕西警告，开发者不能再用舊思維把AI當作現有流程的加速工具，而是要意識到AI帶來了全新的可能性，很多現有應用和代碼在3.0時代都將是冗餘的。

**三、未來架構：神經網絡主導計算機**

卡帕西更大膽地預測，神經網絡將徹底主導計算機，現有計算架構將完全反轉：

*   **格局反轉：** 未來神經網絡將成為宿主進程（Host Process），而CPU退化為協處理器（Coprocessor），僅負責處理確定性的輔助任務。
*   **算力趨勢：** 用於神經網絡的智能算力將佔據全球總算力的絕對核心。
*   **交互界面：** 甚至連交互界面都可能不再是固定的UI設計，而是由擴散模型（Diffusion Models）實時渲染專屬的即時交互界面。

**四、AI能力的“鋸齒狀”缺陷與可驗證性**

卡帕西發現，當前AI的自動化能力表現極度不均衡，呈現出明顯的“鋸齒狀”：

*   **核心原因：** “可驗證性”（Verifiability），再疊加前沿AI實驗室的預訓練數據分佈和強化學習環境設計。
*   **定義：** 傳統計算機自動化“可明確定義”的事物，而新一代大模型自動化“可驗證”的事物。
*   **高可驗證性領域：** 在代碼、數學、邏輯推理等領域，驗證機制清晰，獎勵函數明確，AI的能力極強，甚至超越人類專家。
*   **低可驗證性領域：** 在缺乏明確驗證機制、偏向常識、生活決策和模糊判斷的領域，AI表現粗糙，如早期模型數不清字母，或建議“走路去洗車店洗車”。
*   **本質解釋：** AI模型是統計模擬回路，缺乏生物智能的內在動機、好奇心和自我驅動，不會像人類一樣通過實踐形成常識。其表現好壞取決於實驗室的數據灌輸和強化學習環境設計。

**對創業者的建議：**

*   如果身處可驗證性強的領域，即使前沿實驗室未聚焦，也可自主構建強化學習環境和樣本案例，為模型做微調（Fine-tuning），這將帶來巨大紅利。
*   理論上所有事情皆可自動化，只是難易程度差異。高可驗證性領域將以超乎想像的速度被AI重構。

**五、從“氛圍編程”到“智能體工程”**

卡帕西提出了取代氛圍編程的全新概念——智能體工程（Agentic Engineering）。

*   **氛圍編程（Vibe Coding）：** 旨在抬高每個人的軟件開發能力底線，讓不懂代碼的普通人也能憑直覺、用自然語言描述需求，讓AI生成可用軟件。其價值在於降低創作門檻。
*   **氛圍編程的局限：** 只追求“做出來”，不追求“做得好、做得穩、做得安全”，無法滿足專業生產環境的質量要求。
*   **智能體工程（Agentic Engineering）：** 一門更嚴謹、更專業的學科，旨在“守住上限”，在保持專業軟件開發質量標準、架構品位和邏輯自洽的前提下，實現極致的效率。
    *   **開發者角色：** 99%的時間不再直接寫代碼，而是統籌智能體、監督智能體及把控全局。智能體是強大但帶有隨機性的“AI實習生”，負責所有瑣碎工作，而人類必須負責頂層設計、質量把關和邏輯校驗。
    *   **影響：** 將徹底顛覆“10倍工程師”的概念，精通智能體統籌、系統設計和質量把控的開發者，效率將超越傳統10倍工程師無數倍，可頂得上一個中小型開發團隊的產能。

**AI原生開發者與傳統開發者的區別：**

*   不在於年齡、學歷或經驗，而在於是否“極致榨取工具的價值”，能否深度優化開發環境。AI原生開發者會將各種智能體工具的所有特性用到極致，持續調優自己的開發環境、提示詞體系和智能體工作流。
*   招聘邏輯也將改變：未來的招聘應直接拋給候選人大型實戰項目，考核其駕馭智能體、把控系統質量和解決實際問題的能力，而非死記算法知識點。

**六、AI時代人類不可替代的核心價值：理解力**

卡帕西給出了人類不可替代的核心價值：

*   **人類必須掌控：** 系統規格、高層架構、審美判斷和邏輯正確性。
*   **可交給AI的瑣碎工作：** API細節、代碼填充、語法調試和文檔整理。他自己已不再死記PyTorch、NumPy、Pandas之間的API差異。
*   **MenuGen真實Bug案例：** 智能體曾試圖通過比對郵箱地址來分配用戶額度，而非設計持久的用戶ID，導致嚴重邏輯漏洞。這暴露出智能體缺乏全局邏輯和產品思維。人類的價值在於拍板必須用唯一用戶ID來關聯數據，把控系統頂層設計。

**LPM 1.0：AI虛擬角色交互體驗的突破性技術解析**

**摘要：**
本文稿介紹了一項在AI虛擬角色交互體驗領域取得突破的技術——LPM 1.0（Large Performance Model，大型表演模型）。該模型由米哈游創始人蔡浩宇旗下的AI公司Anuttacon研發，旨在徹底解決困擾業界多年的“表演三難困境”（高表現力、實時推理、長時身份穩定性）。LPM 1.0是全球首個從系統層面破解此困境的視頻生成模型，能讓虛擬角色像真人般自然說話、傾聽、反應與流露情緒，同時保持長期穩定的身份特徵，並支持低延遲的實時流式交互，有望成為對話智能體、直播虛擬形象、遊戲NPC的核心視覺引擎。

**一、LPM 1.0 的核心定位與貢獻**

*   **核心定位：** 基於視頻的角色表演模型，其核心目標是打造一個能學習真人表演行為的視覺引擎。
*   **五大核心貢獻：**
    1.  **系統突破：** 提出全球首個支持單人全雙工對話表演的視頻生成系統，同時滿足高表現力、實時推理、長時穩定三大需求。
    2.  **數據創新：** 構建大規模高質量多模態以人為中心數據集，通過嚴格過濾、音視頻配對及身份感知多參考提取，規模化學習真實對話表演行為。
    3.  **模型研發：** 研發170億參數的Base LPM基礎模型，基於擴散Transformer架構，聯合學習語音驅動動作、傾聽反應、文本控制與多參考身份保留能力。
    4.  **蒸餾框架：** 創新多階段自迴歸蒸餾框架，將高精度的Base模型蒸餾為支持實時交互的Online流式模型。
    5.  **評估標準：** 打造全球首個交互式角色表演基準LPM-Bench，實現對話自然度、可控性、情緒表演、長時一致性的系統化評估。

**二、數據構建：一切的基礎與核心突破**

LPM 1.0的數據集專為對話表演設計，其構建流程分為四大核心模塊：

1.  **高質量以人為中心視頻制備：**
    *   收集涵蓋多種場景、人物、情緒的大規模視頻語料。
    *   經人工審核確保質量與內容相關性。
    *   通過四階段嚴格過濾流水線（場景檢測、人體檢測、人工+模型聯合篩選瑕疵、按人數分類），原始數據保留率低於10%，最終得到3100萬個可訓練片段（2300萬說話、500萬傾聽、300萬閒置）。

2.  **對話音視頻數據處理（幀級三狀態創新）：**
    *   **創新點：** 首次實現幀級“說話”、“傾聽”、“閒置”三狀態標注，捕捉自然對話中傾聽時的細微反應。
    *   **技術實現：** 優化LR-ASD主動說話人檢測模型為幀級三狀態分類器，結合語音檢測與音視頻同步性。
    *   **多人處理：** 設計角色與音頻分離分配模塊，通過人體追蹤、幀級重排序，為每個單人片段配對專屬音頻。
    *   **語義驗證：** 基於Qwen3-Omni微調的音視頻理解模型進行語義驗證（F1值78.37%），有效剔除虛假或非對話傾聽，保證數據純粹性。

3.  **音視頻理解與標注（細粒度動作情緒控制）：**
    *   利用音視頻理解模型為每個片段生成自然語言描述和結構化標籤，覆蓋動作、情緒、環境、鏡頭、表情等多維度。
    *   **數據平衡：** 由於傾聽片段稀缺且多為中性，專門篩選47萬個含清晰情緒反應的傾聽片段，對情緒、表情、能量、動作四維度進行重平衡（下採樣中性、上採樣稀有行為），並保留關鍵時序模式（情緒對比、表情振蕩），讓模型學習豐富的傾聽反應。
    *   **傾聽標注：** 僅保留外觀、性格等不變屬性，直接學習音頻到視覺反應的映射。

4.  **身份感知參考圖像流水線（破解長時身份漂移）：**
    *   **問題：** 傳統單張參考圖像導致模型“腦補”，長時間生成易不一致。
    *   **LPM 1.0方案：** 創新設計三類型多粒度參考圖像，為角色提供完整身份先驗。
        *   **全局外觀參考：** 從長視頻中隨機採樣不同幀，錨定角色整體外貌、著裝、場景。
        *   **多視角身體參考：** 基於SMPL 3D人體模型和GVHMR，計算人體與相機角度，提供多角度外觀信息，解決視角變化後的身份漂移。
        *   **面部表情參考：** 從高清視頻中提取8種基礎表情，驗證準確性，讓模型學會角色專屬的表情細節。
    *   **效果：** 三類參考結合，實現從任意角度、任意表情還原角色身份，徹底告別腦補導致的不一致。

**三、核心模型設計：Base LPM與Online LPM**

整個模型分為高精度、離線生成的Base LPM基礎模型，以及通過蒸餾實現實時、無限長交互部署的Online LPM流式模型。

1.  **Base LPM 基礎模型（170億參數DiT）：**
    *   **核心架構設計：**
        *   **多模態條件注入：** 針對說話與傾聽音頻流，創新採用“交錯式雙音頻注入策略”——偶數層Transformer注入說話音頻，奇數層注入傾聽音頻。獨立的鍵、值、輸出投影參數，最後與文本交叉注意力結果融合。此設計對齊動作分佈、提升參數與計算效率50%、並實現更精準的時序對齊。
        *   **多參考身份條件注入：** 將所有參考圖像編碼為與視頻潛變量同空間的patch token，直接拼接到視頻token序列末尾，參與自注意力計算，實現視頻內容與參考圖像的雙向細粒度信息交互。為不同類型參考分配3D RoPE偏移量區分語義，實現多視角一致性、動態表情保真與長視頻（10分鐘內無漂移）穩定輸出。
    *   **訓練策略：** 採用多階段流匹配（Flow Matching）方法。
        *   第一階段：多模態條件對齊（說話音頻→傾聽音頻→融合）。
        *   第二階段：身份一致性保留（全程全局參考，30%片段加入多視角與表情參考）。
        *   第三階段：時序擴展訓練（分塊自迴歸生成，隨機丟棄真實潛變量解決訓練推理不匹配）。
        *   第四階段：DPO（人類偏好數據優化，解決肢體畸變、口型同步誤差、傾聽僵硬等問題）。

2.  **Online LPM 流式模型（實時、無限長部署）：**
    *   **必要性：** Base LPM為離線模型，無法用於直播、實時對話等需因果推理、低延遲、無限長穩定的交互場景。
    *   **核心設計：** 解決流式控制信號的訓練與推理不匹配，通過“骨幹模型 + 精煉器”架構分離軌跡與重見細節，破解自迴歸rollout的誤差累積。
        *   **流式音視頻輸入適配：** 音頻採用重疊感知分塊音頻編碼（3秒窗口，1秒步長），並用60萬流式數據微調模型。文本控制採用增量更新。
        *   **Online LPM架構：** 將雙向DiT蒸餾為“因果骨幹DiT + 因果精煉器DiT”的兩階段架構（基於DMD框架）。骨幹模型負責穩定時序錨定（基於帶噪歷史KV緩存），精煉模型負責高精度細節恢復（基於乾淨歷史KV緩存）。兩者均採用分塊因果注意力。
    *   **訓練策略：** 四階段課程蒸餾逐步降低優化難度，縮小訓練推理差距。
        *   第一階段：基於常微分方程的初始化。
        *   第二階段：off-policy分佈匹配蒸餾。
        *   第三階段：on-policy分佈匹配蒸餾（關鍵，學會修正自身預測誤差）。
        *   第四階段：精煉器蒸餾（修正骨幹殘留誤差）。
    *   **推理：** 採用滑動窗口解碼，實現實時與無限時長生成。固定大小時序窗口、緩存歷史KV狀態、保留少量Sink token作為注意力錨點，即使訓練僅用5秒片段也能無限時長rollout生成。

**四、基礎設施優化與部署**

LPM 1.0在訓練、在線系統和部署全鏈路進行了基礎設施優化。

1.  **訓練優化（基於TorchTitan框架）：** 數據加載與負載均衡（VAE、音頻、文本嵌入預計算，GPU負載平衡）、混合並行策略（PyTorch FSDP v2，Ulysses上下文並行）、內存高效激活管理（算子級選擇性檢查點，CPU異步卸載）、高效注意力與內核優化（FlashAttention、CuTe、Triton、FlexAttention）。
2.  **在線交互視頻系統（LPM 1.0落地核心）：**
    *   滿足低延遲連續生成、狀態自適應響應、長時一致性三大要求。
    *   **三大架構設計：** 流水線推理執行（生成器、精煉器、VAE三階段并行），多模態狀態自適應控制（預熱、閒置、傾聽、響應間低延遲切換），高效上下文管理（3塊Sink token+2塊滑動窗口混合緩存）。
    *   可對接音頻轉音頻（A2A）模塊，實現全雙工實時對話交互。
3.  **部署端優化：** 模型推理（融合內核、高效注意力、torch.compile）單GPU每塊推理時間低至0.35秒（滿足24幀實時流），系統級（狀態分離、邊界對齊更新、受控前瞻）實現對中斷、角色切換的快速響應。

**五、LPM-Bench：評估基準與模型表現**

1.  **LPM-Bench（全球首個交互式角色表演基準）：**
    *   填補行業空白，涵蓋1000個測試用例，覆蓋說話、傾聽、對話、多樣化人體動作、角色泛化五大場景。
    *   從外觀、表演、音頻三維度保證多樣性，含10%長時用例（3分鐘至1小時）。
    *   **四大評估維度：** 動作動力學、身份一致性、文本可控性、音視頻同步。
    *   評估方式：成對好壞相同（GSB）對比與1-5李克特量表絕對評分。

2.  **模型表現：**
    *   **Base LPM (720P) vs. 頂尖模型：** 人類偏好率64.3%（遠超Kling-Avatar-2的24.4%），在身份一致性、文本可控性上優勢最明顯。核心優勢是能生成分鐘級穩定視頻（對比模型最多30秒），且動作、表情、身份一致性全面領先。
    *   **Online LPM (480P) vs. 實時生成模型：** 人類偏好率82.5%（遠超LiveAvatar的12.5%）。在動作動力學、文本可控性、音視頻同步上全面領先，證明蒸餾未犧牲核心能力，反而提升了實時部署的穩定性。

**六、總結與展望**

1.  **核心突破總結：** LPM 1.0證明了“表演三難困境”可通過數據、模型、蒸餾、部署的系統級協同設計解決。它是第一個能還原真人對話表演的視覺引擎，讓虛擬角色真正成為參與者。
2.  **當前局限：**
    *   交互僅支持單人面向鏡頭的場景，弱依賴社交、物理、敘事環境。
    *   不支持長時對話記憶、多人交互、動態環境中的行為、任意視角的3D一致性。
3.  **未來研發方向（“統一的演員模型”）：**
    *   **時序維度：** 加入對話級記憶、角色人格持久化。
    *   **社交維度：** 支持多人交互、眼神分配、群體話輪轉換。
    *   **物理維度：** 讓角色行為與場景幾何、物體、接觸交互結合。

**意義：** LPM 1.0的出現標誌著AI虛擬角色正式從“畫面走向表演”，從“工具走向伙伴”，它破解的不僅是技術上的三難困境，更是虛擬角色交互的體驗瓶頸。未來，LPM技術有望支撐遊戲NPC、直播虛擬人、AI助手實現更自然、擬人化的交互體驗。

**Box CEO 亞倫·利維：企業AI落地「終極白皮書」核心洞察整理**

**發言人：** 亞倫·利維 (Aaron Levie)，企業雲服務巨頭Box聯合創始人兼首席執行官。
**背景：** 掌舵Box近20年，服務全球1400萬付費用戶，涵蓋絕大多數《財富》500強企業。其在全球頂級科技播客20VC的專訪，被譽為企業AI落地的「終極白皮書」。
**核心判斷（顛覆性觀點總覽）：**

*   AI不會消滅工作，反而會催生數百萬新崗位。
*   軟體價值全面轉移到API，前端界面（UI）不再是核心競爭力。
*   算力預算將從IT資本支出轉向運營支出。
*   智能體既是企業安全的巨大風險，也是唯一的解決方案。
*   未來企業AI生態將是多模型並存，絕不會被單一廠商綁定。

---

**詳細觀點梳理：**

**一、 AI競爭的本質：一場漫長的商業與經濟馬拉松**

*   **駁斥短期衝刺論：** 亞倫·利維明確反對將AI視為決定生死的短期競爭，指出這本質上是一場持續迭代、永無止境的商業與經濟馬拉松，而非一蹴而就的生死對決。
*   **重塑升級認知：** 企業軟體升級是一個耗時多年的浩大工程，真正的關鍵在於如何將模型能力平穩、合規、高效地嵌入業務流程，並持續迭代系統、組織、人才與預算機制，而非誰先獲得最強模型。
*   **AI的增強而非取代：** 認同黃仁勳的觀點，AI的核心價值是「增強」人類工作，而非「取代」。人類永遠處於工作閉環中，只是介入環節和方式改變，不應因誇大AI消滅工作而傷害下一代。

**二、 AI與未來就業：非消滅，而是創造數百萬新崗位**

*   **顛覆性結論：** AI不會大規模消滅律師、工程師、文員、客服等知識工作者，反而會讓很多崗位數量大幅增加（例如未來律師數量會更多）。
*   **價值釋放於傳統行業：** 科技行業僅佔全球GDP約10%，而剩餘90%的製造、金融、醫藥、農業、法律、醫療等傳統行業極度缺乏工程能力。AI的真正價值在於將「硅谷級別的研發能力」普及到這些行業，激發前所未有的需求，從而創造更多崗位。
*   **案例分析：**
    *   **法律行業：** AI雖能快速生成法律文件，但審閱、法院審批、專利申請等實質性法律程序仍需人類律師，瓶頸將轉化為「合格人類律師數量不足」。
    *   **醫療轉診：** AI可自動化預約溝通流程，但若專家號仍需等18個月，瓶頸依然是醫生數量和醫療機構承載能力。自動化只是暴露了原被低效掩蓋的核心瓶頸，需靠增加人力解決。
*   **結論：** AI將人類從重複性、低價值勞動中解放，去解決更核心、稀缺、需專業判斷的問題。初級崗位任務結構改變，但高端專業和複合型崗位將大幅增加。

**三、 未來五年新興職業：智能體操作員（Agent Operator）**

*   **崗位預測：** 未來五年將出現50萬至100萬個「智能體操作員」崗位，成為企業標配。
*   **角色定位：** 這不是普通的提示詞工程師，而是懂技術、懂業務、懂AI、懂流程的複合型人才。他們需深度理解MCP、CLI，會編寫Skills、管理智能體配置文件，並深入各業務團隊。
*   **核心使命：** 為AI智能體「重新設計業務流程」，而非為人類設計流程，以推動技術與業務深度融合，並持續維護與迭代智能體工作流。
*   **價值關鍵：** 在數據高度碎片化、流程固化、合規要求高的大型企業，智能體操作員能將為人類設計的流程改造為適合智能體執行的流程，並進行數據治理、系統集成和變革管理，保障智能體穩定性。

**四、 產業革命：軟體範式重構，Headless與API為王**

*   **價值轉移：** 這是對軟體行業影響最深遠的判斷。過去軟體「UI優先」，使用者是人類，價值與界面體驗綁定；智能體時代，主要使用者是AI智能體，它們僅通過API通信。
*   **Headless成為主流：** 前端界面不再重要甚至可消失，核心能力全部沉澱在後端與API中。軟體的核心壁壘不再是UI體驗，而是API的穩健性、實用性、獨佔性，以及其內嵌的深度業務邏輯與合規能力。
*   **API即價值：** 以前軟體賣的是界面和操作體驗，未來賣的是API能力與業務邏輯。例如ERP的價值是API裡封裝的供應鏈自動化、財務記賬規則等。
*   **Box的實踐：** Box從創立之初就堅持API優先，目前API調用量已遠超人類用戶交互量，智能體的出現加速了此趨勢。API健壯且深入業務的軟體，價值將因智能體普及而大幅提升。

**五、 智能體時代的網絡安全海嘯：風險與解方並存**

*   **雙重風險：**
    1.  **AI生成代碼失控：** AI（如GitHub Copilot）生成代碼規模遠超人類審查極限，可能自主開放端口、修改配置、調用權限，引入災難性安全缺陷。
    2.  **AI攻擊者：** 攻擊者也利用AI以史無前例的速度掃描漏洞、發起攻擊，傳統防護手段難以跟上。
*   **解決方案：** 智能體引發的問題，最終需靠智能體解決。用智能體審查代碼、檢測漏洞、監控異常、響應威脅將成標配。「智能體安全」（Agentic Security）將成為巨大的市場機會。

**六、 企業預算機制顛覆性變革：算力從資本支出轉為運營支出**

*   **算力成本激增：** 未來企業在算力上的支出比例可能接近員工薪資的一半，成為必須投入的核心成本。
*   **預算歸屬轉變：** AI算力（Token預算）將從僵化的IT「資本支出」（CapEx）徹底轉向「運營支出」（OpEx），與行銷費用、營運費用等同。
*   **影響：** 業務線負責人可直接決定算力投入，企業科技支出佔比有望從10-12%提升至20%。
*   **算力分配模式：** 出現「Token最大化」、「風投式預算分配」及「分層配額管理」（如核心員工享無限算力，普通員工享基礎選項）。
*   **核心意義：** 算力已成為核心生產資料，預算歸屬的改變意味著AI真正融入業務核心。

**七、 強監管行業的落地現實：人類永遠是最終責任人**

*   **駁斥完全自動化：** 在金融、法律、醫療等高監管領域，AI可以生成方案、分析數據，但最終的決策、審批、責任承擔必須由人類完成，監管機構不會接受AI做出的最終決定。
*   **專業服務的黃金十年：** 專業服務和變革管理公司（如埃森哲）將迎來龐大增長，因為大型企業需要專業團隊協助改造流程、治理數據、集成系統、培訓人員、建立合規體系。
*   **最大挑戰與價值：** 大多數企業數據不兼容智能體且碎片化嚴重，解決這些問題需要大量的系統升級、數據清洗、流程重構、變革管理。這不是純粹的技術問題，而是組織、合規與管理問題，也是AI落地最難、最有價值的部分。

**八、 未來行業格局：企業AI生態必是多模型並存**

*   **百模大戰長期持續：** 企業絕不會容忍被單一模型供應商綁定，就像雲計算市場多巨頭並存一樣，OpenAI、Anthropic及中國開源模型等將長期共存。
*   **混合使用策略：** 企業會根據場景、成本、合規要求，混合使用不同模型。硅谷現實是大量企業用前沿模型設天花板，用中國開源模型逼近效果。
*   **藍海機遇：** 能夠無縫兼容多種大模型、提供智能體可觀測性、評估能力的平台將掌握巨大價值，成為未來的藍海賽道。

**九、 上市公司、CEO與行業估值的現實挑戰**

*   **軟體業最艱難時刻：** AI技術迭代速度前所未有（一周多次更新），信息價值藏於一線工程師，傳統媒體難以跟上。CEO必須深度扎根生態，否則迅速落後。
*   **市場估值被低估：** 許多軟體公司估值被過度拋售，市場像鐘擺一樣極度低估。
*   **企業分化：** 未來一兩年，抓住Headless、API、智能體的公司將會起飛；未轉型的將被迫重構。例如Atlassian被過度拋售，工程自動化基礎設施仍是優質賽道。
*   **CEO準備度：** 部分創始人和技術型領導者敏銳且深度參與AI實踐；許多傳統管理者缺乏技術儲備，需快速補課。AI浪潮如同歷史所有浪潮，有贏家有輸家。

**亞倫·利維的核心總結與展望：**

*   軟體Headless的速度將遠超預期，平台必須Headless優先。
*   他仍會重倉所有前沿模型的融資輪，因為真正的金礦在於能將硅谷前沿創新落地到企業真實工作流的公司。
*   在智能體可觀測性、評估、垂直行業AI等領域，將誕生大量創新。
*   堅信AI的終極方向是賦能人類、放大能力、普及生產力，讓每個行業都獲得硅谷級的工程能力。
*   這不是一場零和競爭，而是一次全球經濟的升級。未來五年，所有企業、軟體、崗位、預算都將被新的邏輯重塑。

## **AI 编码模型市场深度分析：SemiAnalysis 万字报告解读**

### **引言**

自今年2月5日 Claude Code 迎来发展拐点以来，AI 编码模型市场仿佛被按下了加速键。各大厂商和顶尖实验室每周都有新的编码专用模型问世，如 Claude Opus、GPT-5.5、DeepSeek V4 等，令人目不暇接，也使许多开发者难以分辨其优劣。

SemiAnalysis 的资深分析师经过数月亲手实测所有主流头部模型，包括获得 GPT-5.5 Alpha 测试资格，并深度对比了 Opus 4.7 和 DeepSeek V4 的真实表现，发布了一份长达万字的实测报告。該報告不僅揭示了模型性能，還深入剖析了行业基准测试的“猫腻”、模型定价逻辑及市场竞争格局。

### **报告核心结论**

1.  **OpenAI 回归前沿：** GPT-5.5 发布后，OpenAI 终于重新回到 AI 编码领域的前沿梯队。
2.  **Anthropic 主导地位：** Anthropic 凭借 Claude Opus 系列，依然牢牢占据市场主导地位。
3.  **DeepSeek 技术亮眼但差距拉大：** 开源阵营的 DeepSeek V4 技术虽出色，但已难以撼动闭源大厂的优势，开源与闭源的差距正在重新拉大。
4.  **基准测试失去参考价值：** 所有厂商吹嘘的基准测试分数，正在彻底失去实际参考价值。
5.  **成本效率成核心指标：** 按任务计费的**成本效率**，才是衡量编码模型的真正核心指标。

### **三款代表性模型深度拆解**

#### **一、 OpenAI GPT-5.5：颠覆者归来**

*   **核心特性与性能：**
    *   基于代号为 "Spud" 的全新预训练框架，是 OpenAI 自 GPT-4.5 失败后首次意义上的预训练规模升级。
    *   已全面融入 SemiAnalysis 团队的日常工作流，在部分任务上表现甚至**实质性超越**市面上所有其他模型。
    *   纠正误区：所谓“10万颗 GB200 NVL72 集群上完成训练”，仅指后训练阶段的强化学习（RL），**预训练核心阶段并未达到此规模**，系宣传文字游戏。
*   **定价策略：**
    *   API 定价：每百万输入 token 5 美元，输出 token 30 美元，是 GPT-5.4 的两倍，**略高于 Opus 4.7**。
    *   **优先级服务：** 价格是标准版的 2.5 倍，提供保守且具体的服务等级协议（SLA），例如保证 99% 时间内每秒输出超过 50 个 token，稳定性远高于“快速模式”。
*   **其他版本：**
    *   **GPT-5.3-Codex-Spark：** 专为 Cerebras 芯片设计的蒸馏版 GPT-5.3，非真正的 GPT-5.5 系列。
    *   **GPT-5.5 Pro：** 面向科研场景，仅通过 ChatGPT 和 API 提供服务，主打科学研究和长程推理任务，定价更高，未来有望在科学发现领域有更多成果。
*   **技术亮点：Token 效率**
    *   模型卡片明确标注：**基准测试分数高于 GPT-5.4 的同时，Token 消耗量更低**。
    *   这至关重要：衡量模型定价的核心指标已不再是每 token 价格，而是**完成单个任务的总成本**。Token 效率将成为 2026 年 AI 模型竞争的核心关键词。

#### **二、 Anthropic Claude Opus 4.7：稳定但有隐忧**

*   **整体评价：**
    *   作为 Opus 4.6 的直接替代版本，属于**小幅升级**，基准测试分数有所提升，使用体验稳定，但**缺乏颠覆性技术突破**。
    *   工程师反馈：**勉强接受**，主要原因在于 Opus 4.7 **至今未推出快速模式**，而速度已成为编码工具的核心需求。
    *   痛点转移：工程师的吐槽点已从功能测试是否通过，转变为代码风格、实现思路、架构设计和 Token 效率。
*   **核心功能更新（五点）：**
    1.  **新增高分辨率图像支持：** 可通过截图完成前端样式开发，大幅提升前端开发效率。
    2.  **推理强度新增“xhigh”档位：** 提供更精细的模型推理耗时和能力控制。
    3.  **默认隐藏思考过程 Token：** 简化输出界面，但用户仍需付费，增加了**成本不透明性**。
    4.  **推出测试阶段任务预算功能（API 独享）：** 允许用户建议任务效率阈值，与最大输出 Token 限制不同。
    5.  **全新的分词器：** 提升性能，但导致 **Token 使用量最多增加 35%**，相当于直接涨价 35%。
*   **模型行为调整：**
    *   Opus 4.7 默认减少了工具调用的次数，更依赖自身推理完成任务。SemiAnalysis 团队认为此调整不佳，复杂任务需手动调高推理强度。
*   **历史问题：Opus 4.6 性能 Bug 事件**
    *   在 Opus 4.7 发布前，大量用户指责 Anthropic 故意降低 Opus 4.6 性能，虽遭否认，但工程师普遍感觉异常。
    *   Opus 4.7 发布一周后，Anthropic 发布事后分析报告，承认 3 月至 4 月期间存在三个影响所有 Claude Code 用户的 Bug。
    *   长时间的 Bug 未修复，**侧面暴露了闭源模型的潜在风险**。

#### **三、 开源先锋 DeepSeek V4：工程杰作仍待超越**

*   **版本与参数：**
    *   分为 Pro 和 Flash 两款型号。
    *   V4 Pro：总参数 1.6 万亿，激活参数 490 亿。
    *   V4 Flash：总参数 2840 亿，激活参数 130 亿。
    *   **与闭源模型仍存在明显差距**。
*   **核心突破：百万 Token 上下文**
    *   上下文窗口从 128k 提升至**百万 Token**，所有技术创新都围绕长上下文性能展开。
    *   引入压缩稀疏注意力（CSA）、重压缩注意力（HCA）、流形约束超连接（mHC）三项核心技术。
    *   官方数据显示，在百万 Token 上下文场景下，V4 Pro 的单 Token 推理浮点运算量和 KV 缓存占用大幅降低。
*   **基准测试与表现：**
    *   DeepSeek 自研了智能编码、中文写作等专属基准，认为传统基准无法衡量真实任务能力。
    *   V4 Pro 在其自研任务中能与头部模型抗衡，但在**核心领域仍落后**。例如，高难度中文写作中 Claude Opus 4.7 表现优于 DeepSeek V4 Pro。
*   **技术开源与推理性能：**
    *   开源模型权重、技术报告，并更新了 DeepEP、DeepGEMM 等广泛使用的库。
    *   DeepGEMM 中开源了支持英伟达 GPU 和华为昇腾 NPU 的超级内核，未来可能大量推理任务运行在昇腾芯片上。
    *   推理性能仍有优化空间，首日支持 FP8 精度下，单 GPU 吞吐量约为每秒 150 个 Token。
*   **总结：** DeepSeek V4 是一次极其出色的工程化发布，紧跟前沿水平，是**闭源模型之外成本最低的选择**。但核心能力仍未达到全球顶尖，**开源与闭源的差距正在重新拉大**。

### **Codex 与 Claude Code 实测对比：谁是开发利器？**

SemiAnalysis 团队此前更推崇 Claude，但在 GPT-5.5 Alpha 测试期间，对两款模型进行极限对比后，得出以下结论：

*   **GPT-5.5 (Codex) 的优势：**
    *   在 Codex 插件场景下实现显著提升。
    *   修改代码前会拉取大量上下文信息，擅长**需要深度思考的复杂修改**。
    *   在**代码审查、Bug 排查、代码解释、文档创建与修改**方面表现出色，对代码结构的理解和推理能力更强。
    *   在仪表板数据准确性上远优于 Claude，擅长**复杂数据结构的推理和窄范围高难度任务**。
*   **Claude Code (Opus 4.7) 的优势：**
    *   对用户真实意图的推断能力更强，能更好地理解简洁且不严谨的指令背后的真实需求。
    *   更擅长**开放式、从零到一的全新项目开发**。
*   **新工作流建议：**
    *   先用 **Claude** 完成新应用、新功能的**初始规划和框架搭建**，完成原型开发。
    *   再切换到 **Codex** 解决**具体问题、修复 Bug**。
*   **OpenAI 的生态短板：**
    *   插件和命令行工具功能落后，缺乏百万上下文的快速模式、远程控制、沙盒插件、跨设备会话等功能。
    *   即便 GPT-5.5 模型本身优秀，OpenAI 也需要**加快功能迭代才能追上 Anthropic 的生态优势**。

SemiAnalysis 团队明确指出，**基准测试已无法代表模型的真实实用价值**，但厂商仍乐此不疲地宣传。

*   **通用基准测试的问题：**

    *   **HLE (人类终极考试)：** 题目不代表真实使用场景，存在答案与同行评审文献冲突等问题，但谷歌仍投入巨资优化模型以追求高分。
*   **编码基准测试的问题：**
    *   **SWE-bench (初代)：** 从 Python 仓库自动抓取任务，存在**无人工验证、需求模糊、测试用例不全面、误判**等问题。
    *   **SWE-bench verified：** 虽经人工审核，但超过一半的难题仍存在评估不公，且所有任务来自开源仓库，模型存在**训练数据污染（记忆答案）**问题。OpenAI 已宣布不再上报此成绩。
    *   **SWE-bench pro：** 虽使用更严格授权的仓库并雇佣开发者编写任务和评估，但仍未彻底解决所有问题。
*   **智能体基准测试的问题 (GDPval)：**
    *   虽更贴近真实工作，但仍存在提示词过于明确、缺乏真实场景模糊性、单轮对话无反馈迭代等问题。
*   **厂商的“猫腻”：**
    *   **选择性公布：** 例如 GPT-5.4 发布时未公布基准测试（因被 Opus 4.6 碾压），GPT-5.5 发布时避开 SWE-bench pro，改用自定义的 Expert-SWE 基准（因成绩不如 Opus 4.7 和 Mythos）。
    *   **统一工具套件的误区：** 很多人认为统一工具套件测试更公平，但实际上**工具套件本身就是产品的一部分**，直接决定了任务成本和用户体验。Token 效率与提示词缓存、输入输出比、工具调用模式强相关。

### **【文件整理報告】Google Decoupled DiLoCo：超大规模大模型训练的范式革命**

**整理日期：** 2024年X月X日

**来源：** 最佳拍档频道，大飞解说

---

**一、引言：超大规模大模型训练的“致命痛点”与Google的突破**

*   **背景回顾：** 大约一年前，Google曾发布DiLoCo框架论文。近期，Google DeepMind与Google研究院（由Google首席科学家Jeff Dean主导）联合发布了名为 **Decoupled DiLoCo** 的最新研究论文。
*   **核心痛点：** 当今超大规模大语言模型（LLM）预训练面临的致命问题——集群规模达到数百万块芯片时，硬件故障已是常态，而非偶然。这导致传统同步训练模式下巨大的算力浪费。
*   **痛点数据：**
    *   240万块芯片规模下，传统弹性数据并行的 **有效计算时间仅为40%**。
    *   剩余 **60%的算力** 白白消耗在等待故障修复和集群重配置中。
*   **Decoupled DiLoCo的革命性表现：**
    *   在相同极端故障环境下，将 **有效吞吐率稳定在86%**。
    *   系统几乎实现 **零停机**。
    *   通信带宽需求仅为传统方案的 **60分之一**。
    *   能够整合全球各地老旧芯片和零散算力进行训练，且不影响模型质量。
*   **总结：** 本文旨在深入剖析Decoupled DiLoCo论文，理解其如何解决大模型训练效率与容错性的核心矛盾。Jeff Dean在分布式系统领域的经验再次得到印证。

**二、深度解析：传统SPMD范式的局限性与“效率黑洞”**

*   **SPMD范式（单程序多数据并行）：**
    *   当前大模型训练的主流范式，包括数据并行、张量并行、序列并行等。
    *   核心规则：所有加速器（芯片）必须 **步调一致，完成全局同步**。
    *   致命缺陷：任何设备的卡顿、延迟或硬件故障都会导致 **整个集群停滞**。
*   **规模化下的“概率诅咒”：**
    *   小规模集群下问题不明显。
    *   大规模集群（10万、100万+芯片）下，概率学导致效率崩溃。
    *   **精确计算：** 假设单芯片平均故障间隔时间（MTBI）为1年（工业级顶尖可靠性），但当集群规模达240万块芯片时，整个集群的平均故障间隔时间（MTBF）被压缩至 **不足1分钟**。硬件故障每分钟都会发生，传统训练无法持续。
*   **现有解决方案及其不足——弹性数据并行：**
    *   当检测到故障时，自动缩小集群规模，用健康设备继续训练。
    *   **缺陷：** 故障检测、集群全局重配置、参数重新分片等过程会产生大量 **全局停机时间**。即便开启弹性机制，240万芯片规模下的有效吞吐率仍只有40%。
*   **CAP定理的启示：**
    *   大模型训练的核心矛盾可类比为：一致性（Consistency）、可用性（Availability）、分区容错性（Partition Tolerance）。
    *   **传统SPMD：** 将 **一致性置于首位**（要求所有设备时刻完全同步模型参数），为此牺牲了可用性和分区容错性。

**三、Decoupled DiLoCo的核心革命：范式颠倒与异步解耦**

*   **核心理念：** Decoupled DiLoCo彻底颠倒了传统SPMD的优先级，将 **可用性与分区容错性放在最前面**。
*   **目标：** 在不损失模型质量的前提下，容忍有限的参数不一致，从而实现训练在故障、延迟、异构硬件环境中 **永远不停机**。
*   **技术溯源与演进：** Decoupled DiLoCo是DiLoCo系列技术的第三次迭代。
    *   **DiLoCo (2024, Arthur Douillard团队)：**
        *   核心思路：将全局每步同步改为 **每H步同步一次**。
        *   通过内外层优化器分离，大幅降低通信带宽。
    *   **流式DiLoCo (Streaming DiLoCo, 2025)：**
        *   进一步将模型参数切分成碎片，分步异步传输。
        *   解决了峰值带宽过高的问题。
    *   **前两代方案的短板：** 仍是同步框架，所有学习器必须锁步执行，任一故障或掉队仍会导致系统卡顿。
    *   **Decoupled DiLoCo：** 在Pathways理念支撑下，将间歇性同步彻底升级为 **完全异步解耦**，真正打破了锁步同步的枷锁。

**四、Decoupled DiLoCo架构设计详解**

*   **架构拆分：** 整个框架将传统一体化的SPMD集群拆分为两大核心组件：
    1.  **独立异步学习器 (Learner)**
    2.  **中央同步器 (Syncer)**
    *   两者分工明确，彻底解耦，故障半径严格限制在单个学习器内部。
*   **1. 独立异步学习器 (Learner)：**
    *   **基础执行单元：** 集群被划分为M个独立学习器。
    *   **配置：** 每个学习器持有完整模型副本，分配独立数据分片，运行独立的内层优化器。
    *   **运行模式：** 完全独立运行，不需等待其他学习器。
    *   **三项核心任务：**
        1.  本地持续执行内层优化步骤，处理数据分片，追踪进度。
        2.  在后台 **异步** 向同步器发送本地参数碎片和进度元数据（不影响本地训练）。
        3.  监听同步器更新指令，收到同步后的参数碎片后直接覆盖本地参数，重置进度计数器。
    *   **革命性优势：** **故障完全隔离**。任一学习器故障不影响其他学习器正常训练，从根本上解决SPMD的“一损俱损”问题。
*   **2. 中央同步器 (Syncer)：**
    *   **框架大脑：** 运行在稳定的CPU资源上，负责全局参数的聚合与更新。
    *   **与传统参数服务器的区别：** 彻底抛弃了等待所有学习器同步的逻辑。
    *   **三大关键机制（平衡高可用与高质量）：**
        1.  **最小法定数聚合 (Minimum Quorum)：** 不需等待全部M个学习器上报，达到预设的最小数量K (K≤M) 即可启动参数聚合。故障或掉队的学习器直接排除，保证全局训练不停滞。
        2.  **自适应宽限窗口 (Adaptive Grace Window)：** 达到最小法定数后，不会立刻聚合，而是利用通信与计算重叠间隙，等待一段可控时间，尽可能纳入更多学习器更新，提升同步样本效率，降低外层梯度方差。
        3.  **动态Token加权合并 (Dynamic Token-weighted Merging)：** 根据每个学习器上报的Token数、步数，计算动态权重（处理Token数 * (Token数/步数)），使更新合并更公平，避免快学习器主导或慢学习器拖慢。
    *   **核心算法创新：径向方向平均合并算法 (RDA)**
        *   传统直接平均会导致外层梯度范数随学习器数量增加而衰减。
        *   RDA分别合并梯度的范数和方向，确保合并后的梯度范数不受学习器数量影响，大幅提升超参数稳定性。尤其在学习器数量扩展时，模型性能不衰减。
*   **工程层面的关键优化：**
    *   **平衡张量分片 (Balanced Tensor Fragmentation)：** 优于传统按层分片和张量分片，采用贪心装箱算法将模型所有张量均匀分配到24个碎片中，每一步只传输一个碎片，保证每步通信，同时将峰值带宽降到最低。
    *   **外层梯度压缩：** 验证了将16位bf16梯度压缩到4位int4，模型性能几乎无损，带宽需求进一步降低75%， enabling跨地域、低带宽网络下的训练。

**五、实验验证：碾压性优势与行业颠覆**

*   **验证方法——混沌工程：**
    *   大模型训练领域首次系统性使用混沌工程验证可靠性。
    *   模拟五种核心故障场景（单芯片MTBI、集群总芯片数、芯片速度差异、弹性扩缩容时间、故障恢复时间），通过离散事件模拟器复现百万级芯片集群故障环境。
*   **容错能力验证：**
    *   固定单芯片MTBI为1年。
    *   随着集群芯片数从15万增至240万：
        *   传统无弹性数据并行：有效吞吐率从72% **暴跌至18%**。
        *   传统弹性数据并行：有效吞吐率从88% **降至40%**。
        *   **Decoupled DiLoCo：** 随着学习器数量增加，有效吞吐率稳步提升。
            *   8个学习器：240万芯片规模下 **80%**。
            *   16个学习器：240万芯片规模下 **86%**。
            *   系统正常运行时间接近 **100%**，实现严格意义上的零全局停机。
*   **模型性能：**
    *   在2B、5B、9B稠密模型及2.8B、3.8B激活参数的混合专家模型上进行文本、视觉多任务评测。
    *   结果显示，在极端故障环境下，Decoupled DiLoCo的模型性能（平均分）与传统数据并行训练 **几乎完全一致，无任何性能损耗**。这是其工业级落地的核心前提。
*   **异构硬件适配能力：**
    *   使用TPUv5e和TPUv5p两种不同代际芯片搭建集群，最慢学习器比最快慢18%，并人工注入10%速度差异。
    *   在极端异构环境下，通过最小法定数+自适应宽限窗口，系统计算利用率仍可达 **100%**。
    *   模型下游性能与完全同步的同质集群 **完全一致**。
    *   **意义：** 新老芯片、不同性能硬件可无缝混合训练，大幅降低集群部署成本。
*   **算力拾荒 (Compute Scavenging)：**
    *   传统训练动态加入算力需阻塞全局训练。
    *   Decoupled DiLoCo允许新学习器从邻近学习器 **异步拉取模型状态**，同步过程在后台完成，全局训练完全不受影响。
    *   **实验：** 在固定FLOPs下，动态加入25%、50%、100%、300%临时算力，训练时间分别缩短7%、9%、2%和20%。模型性能不受影响。
    *   **意义：** 彻底盘活全球各地数据中心的闲置算力、潮汐算力、甚至个人设备的零散算力，整合到同一场大模型训练中。
*   **可扩展性：**
    *   稠密模型和MoE模型性能均能对标甚至小幅超越传统数据并行。
    *   随着模型规模和Token预算增加，优势愈发明显。
    *   在1.3万亿Token的超大预算下，其文本和视觉任务平均分已 **超越传统数据并行**，完美印证“越大越好用”特性，适合下一代万亿参数大模型训练。
*   **后训练兼容性：**

### **扩散模型核心概念解析：斯坦福CME 296课程精华总结**

**导语：AI视觉革命的新引擎**

大家好，这里是最佳拍档，我是大飞。

十年前，生成对抗网络（GAN）初露锋芒时，AI生成的图像仍分辨率极低、模糊甚至诡异。而今，AI已能生成分辨率高、色彩丰富，且在光影、材质、物理逻辑上都足以以假乱真的图像与视频。推动这场视觉革命的底层引擎，正是我们今天要深入探讨的——**扩散模型（Diffusion Models）**。

斯坦福大学近期推出了王牌课程CME 296《扩散与大视觉模型》，由阿夫申·阿米迪（Afshine Amidi）和谢尔文·阿米迪（Shervine Amidi）兄弟主讲。他们是法国巴黎中央理工学院的本科校友，分别在MIT和斯坦福深造后，先后在Uber、Google及Netflix担任高级研究员。

阿米迪兄弟在课程伊始便指出，扩散模型领域硬核且数学公式密集，需具备线性代数、概率论、常微分方程及机器学习基础。但请屏幕前的各位不必被吓退，接下来的时间，我将尽量避开枯燥的数学证明，以直观的方式为您呈现扩散模型的核心奥秘。

---

#### **第一部分：扩散模型的哲学与核心思想**

要理解扩散模型，首先需转变思维。

1.  **思维转变：从“无到有”到“去噪”**
    *   人类和早期生成模型倾向于从“无到有”地创作（如画轮廓、上色、加细节）。
    *   扩散模型的思维则完全不同，它秉持着文艺复兴雕塑家米开朗基罗的理念：“雕塑其实早就存在于那块大理石里了，我所做的只不过是把多余的边角料凿掉而已。”
    *   在扩散模型的宇宙中，那块原始的、未经雕琢的“大理石”就是一团**纯粹的随机噪声（Random Noise）**——你可以想象成老式电视机的雪花点。
    *   生成图像的过程，就是在这个纯噪声中，一步一步地**去掉多余的噪声（Denoising）**，最终显露出那只早已隐藏在混沌之中的图像。

2.  **为何要从一团毫无意义的噪声开始？**
    阿夫申给出了三个核心理由：
    *   **易于获取：** 在数学上，从标准概率分布中采样噪声极其简单。
    *   **引入随机性（Randomness）：** 从不同的随机噪声分布开始，即使输入相同提示词，也能得到姿态各异、光影不同的生成内容，保证了**多样性**。
    *   **数学优美性：** 常用的噪声服从**高斯分布（Gaussian Distribution，即正态分布）**，其在数学上具有极其优美且方便的性质，是后续所有推导的基石。

---

#### **第二部分：前向过程——加噪（Forward Process）**

前向过程是扩散模型的第一大核心步骤，旨在教会AI如何识别噪声对图像的破坏。

1.  **过程概述：**
    *   从一张清晰的原始图片 **X0** 开始。
    *   在设定的 **T 个时间步骤**（例如1000步）中，每一步都往图片中注入一点点**高斯噪声**。
    *   这就像往清水里滴墨水，随着时间推移，图片会逐渐变得模糊，最终坍缩成一团纯粹的、毫无特征的标准高斯噪声 **XT**。

2.  **关键技术细节：**
    *   **条件概率：** 定义在给定前一步图像 X_t-1 的情况下，当前图像 X_t 的概率分布。由于添加的是高斯噪声，这个分布必然是高斯分布。
    *   **方差调度（Variance Schedule, Beta_t）：** 一个很小的超参数，随时间步 t 逐渐增大，用于控制每步添加的噪声量。
    *   **方差保留（Variance Preserving）：** 一个巧妙的设计。在加噪声之前，原始图像信号会乘以一个小于1的系数（`根号下(1 - Beta_t)`），防止图像向量数值在多步累积后爆炸，保证整体方差稳定。
    *   **各向同性高斯分布（Isotropic Gaussian）：** 假设每个像素上添加的噪声是完全独立的，互不影响。这极大地简化了计算复杂度，使得在高分辨率图像上训练扩散模型成为可能。

3.  **训练效率优化：直接跳跃**
    *   在训练时，如果想知道第500步的图像，按部就班地计算500次效率极低。
    *   数学家利用高斯分布的叠加性质，通过引入 **阿尔法_t_bar（Alpha bar）**（`1 - Beta_t` 的累乘），推导出一个震撼结论：
    *   给定原始清晰图片 X0，可以直接通过一个公式，**瞬间跳跃到任意时间步 t**，计算出那一刻的模糊图像 X_t。
    *   这个公式同样服从高斯分布，其均值由 X0 决定，方差由 `1 - Alpha_t_bar` 决定。
    *   **意义：** 这使得在训练阶段可以随机抽取时间步 t，并行计算并学习，**极大提升了训练效率**。

---

#### **第三部分：逆向过程——去噪与模型训练（Reverse Process & Training）**

逆向过程是模型的真正考验，即从纯噪声XT一步步倒推出清晰的X0。

1.  **面临的数学挑战：**
    *   目标是求解逆向条件概率分布 `q(X_t-1 | X_t)`，即在给定当前噪声图像 X_t 的情况下，求上一步稍微清晰一点的图像 X_t-1 的概率分布。
    *   这个分布是**不可解（Intractable）**的，因为分母的边缘概率需要遍历所有可能的清晰图像 X0，计算量巨大。

2.  **解决方案：变分推断（Variational Inference）**
    *   既然无法求出精确解，就用深度神经网络去**近似**这个过程，拟合出一个分布 `p_theta`（theta是网络参数）。
    *   通过**詹森不等式**，将无法直接计算的最大化真实图像概率的目标，巧妙地转化成一个可优化的**证据下界（ELBO）**。

3.  **核心突破：引入“上帝视角”**
    *   数学家发现，如果我们在逆向条件概率中引入**“上帝视角”**，即假设我们已经知道了最终的清晰图片 X0，那么 `q(X_t-1 | X_t, X0)` 竟然变得**可解**，并且完全服从一个优美的高斯分布。
    *   这意味着，在训练阶段（我们手头有 X0），我们可以计算出这个“真实”的去噪分布。

4.  **损失函数简化：预测噪声**
    *   ELBO 展开后包含一系列 **KL 散度**（衡量两个概率分布相似度的指标）。
    *   计算两个高斯分布的 KL 散度，在方差固定一致的假设下，等价于计算它们**“均值”之间的均方误差（MSE）**。
    *   **DDPM论文的精彩转化：** 进一步地，研究人员发现，让神经网络去预测那个复杂的均值，在数学上完全等价于让神经网络直接去预测**在第 t 步时，图像里被加进去的那团纯噪声 Epsilon**。
    *   **这是扩散模型训练的最终奥秘：** 神经网络（`Epsilon_theta`）的目标，就是精准预测出原始噪声 Epsilon。

5.  **训练流程概览：**
    *   从数据集中抽取干净图片 X0。
    *   随机选择一个时间步 t。
    *   随机抽取一团噪声 Epsilon。
    *   用前向公式将 Epsilon 加到 X0 上，得到模糊图片 X_t。
    *   将 X_t 和时间步 t 丢给神经网络 `Epsilon_theta`，让它预测噪声。
    *   将预测噪声与真实的 Epsilon 进行对比，计算误差（L2距离），然后反向传播更新网络参数。
    *   **实践优化：** 实验发现，直接去除损失函数中与时间步 t 相关的权重系数，反而能提升最终生成图像的质量。

---

#### **第四部分：模型推理——图像生成（Inference）**

推理过程是模型将学习到的能力转化为实际创作的过程。

1.  **生成起点：**
    *   从标准高斯分布中采样出一个完全随机的纯噪声矩阵 **X_T**，作为生成过程的起点（时间步 T 等于一千）。

2.  **迭代去噪循环：**
    *   将当前噪声图像 X_t 和时间步 t 喂给已训练好的神经网络。
    *   神经网络根据训练经验，预测出当前图像中存在的噪声。
    *   **关键步骤：** 从 X_t 中减去**一部分**预测的噪声，然后**再重新加上一点点随机的高斯噪声**，得到 X_t-1。

3.  **为何要加回噪声？**
    *   这是因为扩散模型本质上是基于**马尔可夫链（Markov Chain）**的**朗之万动力学（Langevin Dynamics）**过程。
    *   神经网络的去噪方向只是宏观指导。如果只是死板地沿着梯度走，容易陷入局部最优解，或偏离真实的图像流形（Image Manifold）。
    *   每次加回的那一点点随机噪声，就像是给模型引入了物理学上的**“热涨落”**，让它保持活力，在图像流形附近游走，防止错误累积。
    *   随着时间步 t 越接近零，加回的随机噪声方差会越来越小。

4.  **最终成果：**
    *   这个“预测噪声、减去部分、再加回一点随机噪声”的循环会严格执行 T 次。
    *   每一次循环，画面都变得稍微清晰一点。当时间步走到零时，所有的随机性被收敛，最终生成一幅前所未见但无比真实的画作。

5.  **效率挑战：**
    *   推理阶段需要神经网络完整运行 T 次（例如1000次），每次都需要大量计算。
    *   这导致早期扩散模型生成一张图片可能需要数分钟，相比GAN模型效率低下。

---

#### **第五部分：性能优化——DDIM（Denoising Diffusion Implicit Models）**

DDIM是解决扩散模型推理效率问题的关键优化方案。

1.  **挑战：** 早期扩散模型（DDPM）在推理时效率低下。
2.  **DDPM的局限：** 严格的马尔可夫过程，即下一状态只依赖于当前状态。
3.  **DDIM的核心洞察：** 研究人员回溯发现，训练损失函数真正起决定作用的是瞬间跳跃的**边缘分布** `q(X_t | X0)`，而不是严格的马尔可夫链式演变。
4.  **解决方案：非马尔可夫前向过程**
    *   DDIM 构造了一个全新的非马尔可夫前向过程，使得每个状态 X_t 都直接隐式地与原始图像 X0 产生联系。
    *   神奇的是，这种改变并不影响最终的训练目标（损失函数与DDPM完全一致）。这意味着可以使用DDPM训练好的老模型，直接切换到DDIM的生成规则。
5.  **关键优化：零随机性与跳跃式推理**
    *   DDIM直接将控制逆向过程随机性的参数 `Sigma_t` **强制设定为零**。这意味着在生成过程中，除了最开始采样的纯噪声 X_T 之外，整个去噪过程将**不再有任何随机性**，变为一个完全确定的**常微分方程（ODE）求解过程**。
    *   由于过程变得确定，且允许状态跨时间步联系，模型可以不再一步步爬行，而是**跳跃式地进行推理**。例如，可以将1000步压缩成100步甚至20步。
6.  **效果与权衡：**
    *   **效果：** 极大地减少了神经网络的运行次数，带来**10倍到50倍的惊人加速**，将生成时间从几分钟缩短到几秒甚至一秒。
    *   **权衡：** 加速是有代价的。当跳跃步数过大时，每次跳跃积累的逼近误差会增大，导致生成的图像质量出现肉眼可见的下降（用FID指标衡量，加速越猛FID越糟糕）。在实际部署中，需在速度与质量间寻找平衡点。

---

#### **总结与展望**

我们从一团毫无意义的马赛克噪声出发，理解了前向过程如何优雅地摧毁图像，随后深入变分推断的腹地，看到晦涩的隐变量概率分布如何在证据下界和KL散度的魔法下，化身为一个直观的、预测纯噪声的均方误差损失函数。最终，我们见证了DDIM如何打破马尔可夫链的枷锁，让模型从缓慢爬行变为轻盈跳跃。

无论数学推导多么复杂，当你凝视那只由代码和矩阵乘法雕刻出的泰迪熊时，你一定能感受到人类智慧在那一刻绽放出的光芒——在混沌的熵增中，找到逆流而上的钥匙，这就是扩散模型的哲学浪漫。

#### **二、 AI浪潮中的角色轉換與新機遇**

*   **李開復的創業新篇章：** 李開復坦言，儘管創新工場投資了13家AI獨角獸（包括英矽智能），但他不滿足於旁觀者角色。當大語言模型興起時，他抓住了機會，創立01.ai，旨在深入企業核心業務，透過AI員工、數位CEO、數位分身等形式，幫助企業（特別是傳統企業）提升效率、增加營收、利潤並加速產品上市。他強調，AI不僅是取代人工、節省成本，更應應用於企業的核心需求。

#### **三、 AI跨越鴻溝進入主流產業的關鍵環節**

*   **行業現狀與潛力：** 李開復認為，企業界普遍已認可AI的生產力價值。
    *   **快速突破領域：** AI在軟體工程領域進展迅猛，01.ai的AI工程師生產力已提升10倍，且未導致裁員。其他數位化、風險低、能創造巨大金融價值的領域（如法律、金融、娛樂等）也將快速採用AI。
    *   **最大價值潛力：** 最需要AI幫助的其實是傳統的實體產業（製造、零售、採礦、農業）。這些行業的自身知識和數據，一旦與AI結合，將成為其最強大的「護城河」，不斷提升AI智能。李開復認為，社會應關注並幫助這些行業，加速AI數據飛輪的轉動。

**一、核心觀點：AI競爭進入 Agent 時代，舊範式已落伍**

*   **範式巨變：** 2026年AI競爭的核心已不再是模型參數量或預訓練數據量。業界正經歷一場比ChatGPT誕生更底層、更徹底的範式地震。

**二、OpenClaw 引爆範式革命：從抵觸到震撼**

1.  **初始抵觸與轉變：**
    *   羅福莉最初對OpenClaw充滿不屑，認為其只是Claude Code套上IM界面，缺乏技術顛覆性。
    *   **轉捩點：** 2026年春節深夜，出於好奇部署測試OpenClaw，從凌晨2點交互至早上6點，經歷了前所未有的興奮與震撼。

2.  **OpenClaw的顛覆性特質：**
    *   **自主性與溫度感：** 能主動感知環境（如時間），表現出高情商，被稱為模型的「靈魂感」。
    *   **效率與賦能：**
        *   **第二天：** 成為能替代生活、工作的「數位分身」，將複雜思考沉澱為體系化Skills。
        *   **第三天：** 僅一兩個小時，就將Agent後訓練的核心研究課題（如多輪交互模擬的User Agent）完整落地，大幅提升科研效率。
    *   **核心價值並非界面：**
        *   **框架能力：** 透過框架彌補模型短板，擁有獨有的「分層、分級的持久化記憶體系」。
        *   **多模型聯合調度：** 無需用戶配置，自動選擇最優模型處理任務。
        *   **激活中層模型：** 即使3B端側小模型，接入框架後也能勝任大模型才能完成的複雜任務，大幅降低成本。
    *   **革命性開源：**
        *   與黑盒封閉的Claude Code不同，OpenClaw完全開源，激活了前所未有的「群體智能」。
        *   小米團隊要求全員每日OpenClaw對話不低於100輪（象徵性），採購Mac Mini降低門檻，兩天內點燃團隊熱情，研發效率提升 **10倍以上**。

3.  **「西蝦東養」現象：**
    *   OpenClaw在中國市場尤為火爆，因國內大量高性價比大模型與開發者對效率的極致需求，以小成本創造巨量生產力。

**三、Agent 時代的 AI 新邏輯與技術方向**

1.  **算力資源分配的徹底反轉：**
    *   **聊天時代：** 研究:預訓練:後訓練 = **3:5:1**（過半算力砸在預訓練）。
    *   **Agent時代：** 合理用卡比例變為 **3:1:1**（預訓練與後訓練投入持平）。
    *   **結論：** 後訓練（尤其Agent場景下的強化學習規模化）才是決定Agent時代模型能力的關鍵，預訓練只是「入場券」。Anthropic的技術路線被證明完全正確。

2.  **程式碼能力的強泛化性：**
    *   **核心抓手：** 程式碼是Agent時代的核心抓手。
    *   **原因：** Agent任務的核心是長程、多輪複雜任務。能支撐128K到1M長上下文的數據集，只有程式碼和書籍，其中程式碼文件關聯度極高、長上下文依賴密集。
    *   **泛化效益：** 做好軟體開發這種超長程任務，能本質提升模型的通用能力，並泛化到金融分析、工業設計、內容創作等所有長程複雜任務。

**四、小米 MiMo-V2 系列模型：Agent 時代的技術剖析**

1.  **戰略佈局：**
    *   小米於2025年底推出MiMo-V2-Flash，以高性價比和推理速度刷新業界認知。
    *   2026年初推出MiMo-V2-Pro、Omni、TTS，完成全場景矩陣佈局，羅福莉稱之為「悄無聲息的伏擊」。

2.  **核心架構選擇：**
    *   **放棄主流MLA：** 羅福莉團隊放棄了當時業界主流的MLA（多頭潛在注意力機制），因其極致優化導致後續優化空間不足，不適合Agent時代。
    *   **獨家架構：** 堅定選擇「**混合注意力機制 (Hybrid Attention) + 多詞元預測 (MTP)**」。
    *   **三大不可妥協要求：**
        1.  **極致長上下文效率：** 能支撐1兆甚至更長上下文，嚴控推理成本。
        2.  **極高推理速度：** 用戶體驗一旦提速便無法回頭。
        3.  **足夠架構靈活性：** 能夠適應快速迭代的Agent框架。
    *   **MiMo架構優勢：**
        *   高稀疏比（Flash 5:1，Pro 7:1），大幅減少KV Cache佔用，降低長上下文推理成本。
        *   MTP技術提升基座能力，倍數級提升推理速度，且因只採納準確詞元故不產生幻覺。
        *   TPS領先（Flash 100-150，Pro 60-100）。
        *   架構簡潔靈活，為後訓練和Agent適配留下巨大優化空間，MLA無法實現。

**五、萬億參數模型訓練與團隊管理：實戰經驗**

1.  **訓練挑戰：**
    *   **非數據與算力堆砌：** 訓練MiMo-V2-Pro前期架構研究與中後期後訓練的算力投入是訓練環節的3-5倍。
    *   **最大難點：** 「數值不穩定性」——損失值(loss)頻繁跳變、激活值異常、專家負載極端不均衡。
    *   **小米態度：** 遇到loss跳變，「立刻停訓排查」問題根源，而非強行壓制，因為loss spike會「打死模型專家，導致參數永久失效」。

2.  **小米團隊模式：**
    *   **平權與去層級化：** 堅信「平權有利於創新」，無固定小組、無職級、無固定發布時間節點，只求模型達到最優狀態。
    *   **人才篩選：** 「環境優於經驗」，不要求大模型經驗，重視研究熱愛，博士比例55%，甚至招募本科生。認為AI研發能力可快速習得，本科生無固化思維對新範式有更大想像力。
    *   **多樣性與靈活性：** 允許預訓練人員轉崗後訓練，帶來數據多樣性。後訓練（RL Infra）要求「容忍模糊性」，需團隊極高靈活性和敏捷性。

**六、AGI、中美差距與未來展望**

1.  **AGI 時間點預測：**
    *   目前已完成AGI歷程的20%，2026年年內將推進到60%-70%，**兩年內實現AGI**。
    *   **核心依據：** AI自訓練、自迭代的臨界點即將到來。AI將自主完成模型訓練、架構設計、演算法優化等核心工作，甚至替代頂尖研究員。

2.  **中美大模型技術差距：**
    *   國內Kimi、MiMo等團隊已具備1T以上基座模型研發能力。
    *   與Claude Opus 4.6的代差僅 **兩三個月**。未來兩三個月是全球AI格局定型的「核心窗口期」。

3.  **長期技術佈局關鍵決策：**
    *   行業不會止步於1T參數。需立刻決定：「到底規模化什麼？」(參數、上下文、數據、Agent協作？) 以及「在什麼晶片上規模化？」這將決定半年到一年後的領先者。

4.  **開源與閉源的終極選擇：**
    *   **開源價值：** 加速AGI落地，支撐海量分散算力，推動Agent框架、晶片、能源全產業鏈協同發展。
    *   **前提：** 企業是否擁有不可替代的戰略生態位。若生態位僅是模型本身則不敢開源，若擁有适配Agent時代的架構、落地場景、全鏈路能力，開源就是最強武器。

**七、AI 時代人類的核心價值**

1.  **AI替代的現實：** 今年內大量工作（從程式設計師到研究者）將被AI替代，滲透所有生產力場景。

## 霍華德·馬克斯沃頓對話核心思想：穿越周期的投資哲學

### 引言

近日，全球知名投資大師、橡樹資本（Oaktree Capital）聯合創始人霍華德·馬克斯（Howard Marks）應邀在其母校賓夕法尼亞大學沃頓商學院，與金融學兼職教授克里斯托弗·蓋齊（Christopher Geczy）展開一場深度對話。此次交流中，馬克斯深入剖析了關於市場情緒、買賣策略、成長與價值辨析以及風險控制的底層投資思維。

本次對話前，主辦方特別要求學生研讀了馬克斯的三篇核心備忘錄：《少一點虧損，還是多一點贏家？》、《什麼才真正重要？》以及《測量市場溫度》。這三份材料共同構成了他一套完整且歷久彌新的投資哲學。本文將結合2026年當前的市場環境，探討這套哲學的實戰應用。

### 一、市場情緒：當前我們身處何方？

霍華德·馬克斯對當前市場的判斷是：**市場情緒已從極度樂觀向負面方向出現明顯擺動，且這種擺動既劇烈又突然。然而，整體市場仍被樂觀情緒所支撐，遠未達到絕望或自暴自棄的程度。**

得出此結論的原因如下：
*   **長期上漲積累的樂觀情緒：** 過去很長一段時間，全球市場持續上行。標普500指數在2023、2024、2025年分別上漲約26%、27%和18%，三年累計漲幅達87%，若從2022年第四季度低點算起，整體漲幅已超100%。這種連續大漲本質上由投資者的樂觀情緒推動。
*   **認知失調現象：** 在樂觀市場中，投資者易出現認知失調，傾向於吸收好消息、忽略壞消息，正如摩根大通CEO傑米·戴蒙所言，人們常會假裝看不見第一隻“蟑螂”。
*   **市場轉向的觸發點：** 情緒鐘擺不會永駐一端，負面因素終將累積。馬克斯梳理了此次市場轉向的幾個關鍵觸發點：
    *   **關稅政策衝擊：** 去年4月，特朗普政府宣布超預期新一輪關稅，導致市場一周內快速下跌約15%。
    *   **AI技術對軟件行業的顛覆：** 今年2月，OpenAI和Anthropic新一代編程模型的發布，直接衝擊傳統軟件公司的核心價值，引發對其巨額信貸債務風險的擔憂。
    *   **私人信貸領域風險暴露：** First Brands和TriColor等公司破產並伴隨潛在欺詐，引發市場對私人信貸放貸標準過低的質疑。
    *   **地緣衝突：** 加劇能源與經濟不確定性，進一步推升投資者擔憂。
*   **人性的極端化：** 現實世界的事態常在“還不錯”和“沒那麼好”之間小幅擺動，但投資者的判斷卻容易從“完美無缺”跳到“毫無希望”。
*   **估值偏高：** 儘管經歷回调，標普500市盈率約22倍，仍高於16-17倍的歷史均值。常見的“這次不一樣”論調，在歷史上多次泡沫中均出現過。

**結論：** 當前市場處於樂觀退潮、謹慎升溫的階段，但遠未達極端低估。

### 二、買入時機：何時才是“抄底”的最佳時機？

**霍華德·馬克斯的結論直截了當：投資中最愚蠢的做法之一，就是非要等到底部再買。**
*   **“底部”的不可預測性：** 所謂底部，是資產價格開始上漲前的最後一天。事後才能知曉，事前無法預判和精準捕捉。追求完美地等待底部，往往會錯失整個行情，眼睜睜看著便宜資產漲回合理價格。
*   **買入的唯一合理依據：** 判斷買入的唯一合理依據是“東西已經便宜了”。這可通過估值、基本面和安全邊際來判斷，而不是去判斷今天是否是下跌的最後一天。
*   **2008年金融危機的實踐：** 在2008年全球金融危機中，當金融體系瀕臨崩潰時，橡樹資本的決策是：如果世界真的熔毀，做什麼都沒意義；如果世界沒垮，則必須出手。他們買入的是被杠杆收購公司最優先級債務，安全邊際極高，即使公司價值僅剩原有四分之一，仍不會虧損。在市場恐慌、價格低到極致時，他們逆勢大舉投入，最終獲得豐厚回報。
*   **逆勢投資的前提：**
    1.  **有錢可投：** 資金需在危機前募集到位，而非危機來臨時才籌集。
    2.  **有膽量去投：** 需具備對抗全場恐慌情緒的勇氣。
    3.  **危機前保持謹慎：** 平時不盲目放大風險，才能在機會來臨時有能力布局。

### 三、投資視角：成長與價值，並非水火不容

霍華德·馬克斯明確否定了將成長投資（買高增長科技股）與價值投資（買低估值傳統股）視為對立關係的機械二元劃分。
*   **歷史誤解：** 20世紀60年代前，股票市場並無成長與價值之分。1962年後，“成長股”概念出現，指未來前景光明的公司。隨後，不投成長股的人自稱為“價值投資者”，才固化了成長=高增長、價值=低價格的錯誤劃分。
*   **核心在於價格與價值的匹配：** 增長快慢不代表投資的聰明與否。關鍵在於**“你為這種增長支付了什麼樣的價格”**。高增長不等於無限溢價，低增長也不代表天然有價值。價格與內在價值的匹配度，才是決定投資成敗的核心。
*   **“漂亮50”泡沫的教訓：** 1969年，市場追捧基本面優異的“漂亮50”成長股，但因價格嚴重高估，即使持有五年，最終也虧損約95%。這證明了任何資產都不可能貴到不能買，也沒有任何資產糟到只要價格足夠低就不能成為好買賣。**“好的投資，不是來自於買的好東西，而是來自於東西買得好。”**
*   **當前科技股投資應用：** 無論是追捧AI巨頭、科技七巨頭，還是完全排斥高成長只盯低估值藍籌，都可能走向極端。正確的做法是，對所有股票都要評估其內在價值，只在價格低於價值時買入，並為增長支付合理的成本。

### 四、賣出策略：何時放手，何時堅守？

市場上關於賣出的討論遠少於買入，大多數投資者的賣出決策受情緒驅動。
*   **情緒化賣出的非理性：** 漲了怕利潤回吐而賣出，跌了怕繼續虧損而賣出，這兩種行為不可能同時合理，完全是被情緒牽引。
*   **理性賣出法則：反向買入。** 假裝自己目前未持有該資產，問自己一個問題：“如果是今天，我還會買它嗎？”
    *   **如果答案是“不會”：** 說明當前價格缺乏吸引力或基本面惡化，應考慮賣出。
    *   **如果答案是“會”：** 則應繼續持有。
*   **“持有”是一種中間狀態：** 當資產價格合理、公司基本面持續改善，既未便宜到非買不可，也未貴到必須賣出時，安心持有就是最佳選擇。
*   **警惕過早賣出偉大的資產：** 馬克斯基於自身謹慎性格，曾錯過巨大收益。他以亞馬遜（Amazon）為例，從1999年泡沫時的90美元跌至6美元，隨後復權漲至3300美元。大多數人會在小幅上漲後賣出鎖定收益，卻錯過了85%以上的複利增長。
*   **賣出核心：判斷是否仍是稀缺的複利機器。** 真正偉大的投資想法極度稀缺，普通人一生遇不到幾個。賣出決策不應盯著短期漲跌，而是判斷資產的核心邏輯是否依然成立，它是否仍是能持續創造價值的標的。只要核心邏輯未變，就不要輕易拋售。

### 五、投資哲學：求勝致勝，抑或少犯錯誤？

投資的終極選擇是：你想靠更多制勝的辦法賺錢，還是靠盡量少犯致命錯誤來盈利？這個問題沒有標準答案，但決定了投資體系的長期可持續性。
*   **網球比賽類比：**
    *   **職業網球：** 贏球者必須主動進攻，打出更多制勝球。
    *   **業餘網球：** 贏球者往往是失誤最少的，穩定把球打回對面，等待對手犯錯。
*   **投資領域的應用：**
    *   **信貸/固定收益投資：** 核心是“少犯錯，避開壞賬”，靠不虧來賺取穩定回報，屬於“贏得輸家的遊戲”。
    *   **風險投資/成長股投資：** 必須抓住幾個“超級大贏家”，以少數制勝之舉來覆蓋成本並獲得超額收益。
*   **選擇適合自己的道路：** 霍華德和橡樹資本選擇的是“少犯錯”的路線，將風險控制置於首位，追求業績的一致性和穩定性。但這並非唯一正確的路徑。投資者必須選擇一條**適合自己的性格、能力圈和風險承受能力**的道路，而非盲目模仿他人。模仿職業選手的業餘選手會頻繁失誤，風險承受能力低的普通投資者盲押高風險項目也可能虧損慘重。

### 總結與啟示

霍華德·馬克斯此次對話的核心內容可濃縮為以下五點，以助記憶和實踐：

1.  **測量市場溫度：** 市場情緒永在極端之間擺動，勿被樂觀或恐慌裹挾，需理性判斷估值高低。
2.  **放棄抄底幻想：** 買入的唯一依據是資產足夠便宜，安全邊際比精準擇時更重要。
3.  **價格匹配價值：** 成長與價值並非對立，好投資不是買“好東西”，而是“買得好”，價格永遠要與價值匹配。

## Cerebras Systems：AI推理芯片的顛覆者與其IPO之路

### 核心摘要

本報告深入剖析了Cerebras Systems這家在AI算力領域挑戰英偉達霸權的公司。Cerebras憑藉其獨特的晶圓級芯片技術，專注於AI推理加速，並在性能上超越英偉達最新旗艦產品。公司已遞交IPO申請，預計2025年實現盈利，估值高達230億美元，並與OpenAI、AWS等巨頭建立戰略合作。報告將從公司概覽、技術壁壘、商業化進程、競爭格局及行業洞察等多維度，呈現Cerebras如何將一個75年未解的技術幻想變為商業現實，並有望改寫AI芯片的全球格局。

### 一、 公司概覽與IPO亮點

*   **公開上市計劃：** Cerebras Systems於2026年4月17日向美國SEC提交IPO申請，計劃在納斯達克上市，代碼為CBRS。
*   **驚人財務預期：**
    *   預計2025年全年營收5.1億美元，淨利潤8790萬美元，實現從巨額虧損（2024年淨虧損4.848億美元）到盈利的逆轉。
*   **高額估值與投資方：**
    *   在2026年2月完成的H輪融資中募資10億美元，估值達到230億美元。
    *   投資方陣容豪華，包括老虎環球、AMD、富達、Benchmark Capital、Altimeter等頂級機構。
*   **OpenAI超級計算協議：**
    *   與OpenAI簽署的計算協議規模可能從超100億美元擴展至300億美元。
    *   OpenAI還將獲得Cerebras最多10%的股權認購權證。這標誌著頂級AI模型公司開始大規模押注非英偉達體系的芯片廠商。

### 二、 技術創新與核心壁壘

*   **專注AI推理加速：** Cerebras的核心定位是AI推理加速賽道，專為大語言模型推理場景做極致優化。
*   **全球最大芯片：** 製造出人類歷史上面積最大的芯片，達到傳統GPU的56倍。
*   **性能卓越：**
    *   CEO安德魯·費爾德曼聲稱，其芯片推理速度比英偉達GPU快數十倍。
    *   獨立機構Artificial Analysis實測（2025年5月，Meta Llama 4 Maverick 400B模型）：Cerebras芯片達到2522 token/秒，約為英偉達最新Blackwell B200（1038 token/秒）的2.4倍。
*   **核心技術判斷：**
    1.  **數據移動是瓶頸：** AI計算的核心矛盾不在於算力不足，而是數據在處理單元之間移動的成本太高。傳統處理器架構不適應AI模型高頻中間結果傳輸。
    2.  **推理内存訪問模式：** AI推理需密集讀取内存。一個70億參數模型生成一個詞，就需要從内存搬運約140GB數據，GPU的HBM内存雖容量大但讀寫速度慢，形成瓶頸。
*   **Cerebras的解決方案——晶圓級芯片與SRAM：**
    *   直接將芯片做得足夠大，可在芯片上集成足夠多的SRAM（44GB）。
    *   結合SRAM的極致速度優勢與大容量，解決容量與速度問題。
    *   數據幾乎不需要離開芯片，大幅降低IO數據傳輸帶來的功耗。
*   **解決晶圓級芯片良率難題：**
    *   晶圓級芯片良率一直是75年來行業公認的難題（缺陷率高）。
    *   Cerebras首次將存储器行業的冗餘策略移植到處理器芯片：將處理器設計成數十萬塊相同的小單元(tile)，缺陷單元直接關閉，預留的冗餘單元自動頂上，完美解決大芯片良率問題。
    *   經過近五年的反覆失敗與重新設計，Cerebras的系統於2019年底成功運行，並在2026年實現晶圓良率超越英偉達同類產品。

*   **共同特性：** 两款模型均原生支持 **100 万 token 上下文**，在开源大模型领域属前所未有的突破。

### 三、 极致效率飞跃：百万上下文与低成本推理

尽管上下文长度和参数规模大幅提升，V4 系列的推理成本反而显著降低：

*   **DeepSeek-V4-Pro：** 在 100 万 token 上下文设置下，
    *   单 token 推理 FLOPs：仅为上一代 V3.2 的 **27%**。
    *   KV Cache 占用：仅为上一代 V3.2 的 **10%**。
*   **DeepSeek-V4-Flash：**
    *   单 token 推理 FLOPs：仅为 V3.2 的 **10%**。
    *   KV Cache 占用：仅为 V3.2 的 **7%**。

**核心价值：** 上下文长度扩展近 8 倍，但单 token 算力需求大幅下降。这意味着超长上下文场景不仅可行，且成本低廉，可大规模商用。

### 四、 三大核心模块同步革新：全维度效率优化

DeepSeek-V4 首次将“稀疏化”理念拓展至 Transformer 架构的注意力机制、残差连接和优化器三大核心模块，实现从参数稀疏到上下文稀疏的全维度效率优化。

1.  **混合注意力架构 (Mixed Attention Architecture)：**
    *   **背景：** 传统自注意力机制计算复杂度随上下文长度平方级增长。
    *   **创新：** 设计全新混合注意力架构，交替使用压缩稀疏注意力 (CSA) 和重压缩注意力 (HCA)，并搭配滑动窗口分支，实现“粗粒度+细粒度，稀疏+稠密”组合拳。
        *   **压缩稀疏注意力 (CSA)：** 先将每 4 个 token 的 KV 缓存合并为摘要，再通过闪电索引器，让每个查询 (query) 从压缩摘要中挑选最相关的 top-k 进行计算。
        *   **重压缩注意力 (HCA)：** 将每 128 个 token 合并为摘要，对所有压缩摘要进行稠密注意力计算，保证长程依赖。
        *   **滑动窗口注意力分支：** 专门处理近距离 token 细节依赖。

2.  **流形约束超连接 (mHC)：**
    *   **背景：** 传统残差连接在深层网络中易出现数值不稳定问题。
    *   **创新：** 将残差映射矩阵约束到双随机矩阵流形上，确保映射矩阵的谱范数不超过 1，从而保证信号传递的非扩张性，提升深层网络训练稳定性。
    *   **额外优势：** 解耦残差流宽度与模型隐藏层维度，以极小计算开销提供新的模型扩展维度，提升表达能力。

3.  **Muon 优化器：**
    *   **背景：** 传统 AdamW 优化器在大规模训练中收敛速度和稳定性有局限。
    *   **创新：** V4 系列在除嵌入层、预测头、归一化层外的核心模块中，用 Muon 优化器替代 AdamW。Muon 优化器具有更快的收敛速度和更强的训练稳定性。
    *   **DeepSeek 优化：** 针对性设计混合牛顿-舒尔茨迭代算法，分两阶段完成正交化（前 8 步快速收敛，后 2 步精准稳定），提升效率和数值稳定性。

### 五、 全栈工程优化

为支撑新架构的落地，DeepSeek-V4 进行了全面的工程优化：

1.  **专家并行 (EP) 细粒度优化：**
    *   将专家拆分成多个波次，形成计算、通信、内存访问完全重叠的流水线内核，将跨节点通信延迟完全隐藏在计算过程中。
    *   在英伟达 GPU 和华为昇腾 NPU 平台均验证，通用推理场景提速 1.5-1.73 倍，强化学习采样场景最高提速 1.96 倍。
2.  **TileLang 内核开发与批量不变确定性内核库：**
    *   采用领域专用语言 (DSL) TileLang 快速开发融合内核，替代原生算子，实现极致运行时性能。
    *   开发端到端、按位一致的批量不变确定性内核库，解决传统算子在不同批次、硬件下输出不一致问题，确保训练稳定性和可复现性。
3.  **FP4 量化感知训练及框架优化：**
    *   MoE 专家权重和 CSA 索引器 QK 路径采用 FP4 (MXFP4) 量化，并设计无损 FP4 到 FP8 反量化流程，复用现有 FP8 混合精度训练框架。
    *   训练框架适配 Muon 优化器、mHC 结构和混合注意力机制，设计两阶段上下文并行策略，解决超长上下文训练难题。
    *   推理框架专门为混合注意力设计异构 KV 缓存结构和磁盘 KV 缓存存储方案，解决百万上下文场景下共享前缀缓存复用问题，降低高频调用成本。

### 六、 训练范式革新："分化再统一"后训练

为解决 V3.2 混合强化学习中不同能力互相干扰的问题，V4 采用两步走策略：

1.  **领域专家专项训练：**
    *   针对数学、代码、Agent、指令跟随等核心领域，分别训练独立专家模型。
    *   每个专家模型先用高质量数据监督微调，再用 GRPO 算法针对领域专属奖励模型强化学习。
    *   根据推理强度训练不同配置专家模型，形成 **Non-think、Think High、Think Max** 三档推理模式，用户可灵活选择。
        *   **Think Max 模式：** 注入强指令，拉满上下文长度和推理深度，释放模型能力上限，适合科研、复杂推理场景。
2.  **On-Policy 蒸馏 (OPD) 整合专家：**
    *   将十多个不同领域的专家模型融合成一个统一模型。
    *   让最终统一模型作为学生，自主生成回答后，匹配最擅长该问题的专家模型输出分布，在 logit 层级完成对齐。
    *   **工程方案：** 将所有教师模型权重卸载到分布式存储，仅缓存每个教师模型最后一层隐藏状态，训练时按教师索引对样本排序，确保 GPU 显存中每次只驻留一个教师模型的预测头。

#### Agent 能力专项优化

*   在后训练阶段将 Agent 作为独立专家方向进行专项训练。
*   工具调用格式从 JSON 替换为带特殊 token 的 XML 格式，降低转义错误，提升稳定性。
